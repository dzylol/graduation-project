# Chapter 0: 准备知识

## 0.1 你需要的前置知识

### 0.1.1 矩阵乘法的几何直观

**标量乘法**是"拉伸"：3 × 2 = 6 把 2 拉伸成 6。

**矩阵乘法**是"旋转+拉伸"：

```
[ 0  -1 ]     [ 3 ]     [ 0×3 + (-1)×4 ]     [ -4 ]
[ 1   0 ]  ×  [ 4 ]  =  [ 1×3 +  0×4  ]  =  [  3 ]
```

这个矩阵把点 (3,4) 旋转 90° 变成 (-4,3)。

> **📌 一句话总结**：矩阵乘法 = 线性变换（旋转 + 拉伸），不擅长处理"弯曲"的变换。

### 0.1.2 导数的物理直观

**导数 dh/dt 的几何含义**：t 时刻的瞬间变化率。

想象你开车：
- t=1小时时，里程表读数 h=60公里
- t=1.01小时时，h=60.6公里

那么：

```
dh/dt ≈ (60.6 - 60) / (1.01 - 1) = 0.6 / 0.01 = 60 公里/小时
```

**dh/dt ≈ 60** 意味着：此刻车速约 60 km/h。

**微分方程 dh/dt = a·h** 说："变化率正比于当前值"。

- 若 a > 0：指数增长（利滚利）
- 若 a < 0：指数衰减（放射性衰变）

> **📌 一句话总结**：导数描述"瞬间变化率"，微分方程描述"变化率与当前值的关系"。

### 0.1.3 复数的几何直观

复数 p = σ + jω 有两个部分：

```
σ (实部)：决定衰减/增长
ω (虚部)：决定振荡频率
```

**几何意义**：

```
复平面
    Im
     |
     |    p = -0.5 + 2j
     |       ↗
     |      /
     |     /  ← 复数 p 是平面上的一个点
     |    /
     |   /
  ---+--------→ Re
     |
```

**复数的模 |p|**：从原点到点的距离，决定信号幅度。
**复数的幅角 arg(p)**：与正实轴的夹角，决定振荡相位。

> **📌 一句话总结**：复数 σ + jω 把"衰减/增长"(σ) 和"振荡"(ω) 打包成一个数。

### 0.1.4 指数函数 e^x 的直观

e ≈ 2.718...

```
e^0   = 1
e^1   = 2.718...  (>1，增长)
e^(-1)= 0.368...  (<1，衰减)
```

**复数指数** e^(jω) 是"旋转"：

```
e^(j·2π) = 1      （转一圈回到 1）
e^(j·π)  = -1     （转半圈）
e^(j·π/2)= j      （转 90°）
```

**关键性质**：|e^(jω)| = 1（纯旋转，不改变模长）。

> **📌 一句话总结**：e^(σ+jω) = e^σ · e^(jω) = 既有伸缩(e^σ)又有旋转(e^(jω))。

# Chapter 1: 动机 —— Transformer 哪里不行？

## 1.1 Self-Attention 的 O(N²) 困境

### 一个具体的数字游戏

假设你有一条长度为 **N=512** 的分子 SMILES 序列（大约 100 个原子的分子）。

Transformer 的 Self-Attention 需要计算一个 **N×N** 的注意力矩阵：

```
注意力矩阵大小 = 512 × 512 = 262,144 个元素

每个元素是一次点积：dot(query, key) → 需要 d_model 次乘法

总计算量：O(N² × d_model) = 512² × 256 ≈ 6700 万次操作
```

如果序列变成 **N=4096**（很长的 DNA 序列）：

```
注意力矩阵大小 = 4096 × 4096 = 16,777,216 个元素

总计算量：O(N² × d_model) ≈ 43 亿次操作
```

**这就是 O(N²) 的恐怖**：序列长度翻 8 倍，计算量翻 64 倍。

### 显存瓶颈

不仅仅是计算，还有**存储**：

```
注意力矩阵 N×N：
N=512:   512²  = 262,144  floats ≈ 1 MB
N=4096: 4096² = 16,777,216 floats ≈ 64 MB  （每层！）

如果有 12 层 Transformer：64 MB × 12 = 768 MB 仅存储注意力矩阵
```

对于科研场景的 **基因组数据（DNA 序列可能上百万 token）**，这直接爆炸。

### 延迟问题

实时预测时，Transformer 必须等待"看到整个序列"才能输出。

而科研场景常需要**流式处理**：数据一点一点来，模型一点一点输出。

> **📌 一句话总结**：Transformer 的 O(N²) 复杂度在长序列上遭遇显存和延迟双重瓶颈，无法高效处理基因组、音乐、超长文本等场景。

---

## 1.2 替代方案的三大失败

过去十年，研究者提出了多种"更高效"的架构来挑战 Transformer：

| 架构 | 核心思想 | 致命弱点 |
|------|----------|----------|
| **Linear Attention** | 用核函数近似 softmax，把 O(N²) 变成 O(N) | **无法做内容选择**——无法区分该记住什么、该遗忘什么 |
| **Gated Convolution** | 用卷积代替注意力，O(N) 但感受野固定 | **感受野是硬编码的**，无法动态调整 |
| **Standard RNN** | 状态更新 `h_t = f(x_t, h_{t-1})`，天然 O(N) | **必须串行**：计算 h_100 必须先算完 h_0...h_99，无法并行 |

### 为什么"无法做内容选择"最致命？

想象处理分子 SMILES：`CC(=O)OC`（乙酸甲酯）

某些 token 是"关键官能团"（决定分子性质），另一些 token 是"填充/噪声"。

**好的模型应该**：
- 记住关键碳链 `CC(=O)O`
- 遗忘填充 token
- 根据内容动态决定保留/丢弃

**Linear Attention 的问题**：
```
注意力分数 = K(x_t) · sum(K(x_i))  （线性，近似）

这里 K(·) 是固定的核函数，输入内容只改变"被加权的值"，
不改变"加权的方式"——即，无法真正选择。
```

> **📌 一句话总结**：Linear Attention、Gated Conv、RNN 都无法实现"根据内容动态选择保留哪些信息"，这是它们失败的根本原因。

---

## 1.3 核心问题：无法根据内容选择保留哪些信息

### 一个思维实验

给你两个序列：

**序列 A**：`CC(=O)OC` （乙酸甲酯，有化学键信息）
**序列 B**：`CCCCCCCC` （8 个碳原子，直链烷烃）

如果模型无法"选择性"：
- 处理 A 时，它必须记住每一个 token，因为化学性质由官能团决定
- 处理 B 时，它只需要记住"这是一串碳"，不需要记住具体哪个碳

**好的模型对 A 和 B 应该有不同的处理策略**——这就是"选择性"的意义。

### Transformer 真的做到了选择性吗？

是的，Transformer 的 softmax attention 理论上可以做到：

```
attention(i, j) = softmax(Q(i) · K(j)) = 选择性地关注任意位置 j
```

**但代价是 O(N²) 的注意力矩阵**。

### Mamba 的目标

> **"我想要 Transformer 的选择性，但我不要 O(N²) 的代价。"**

Mamba 的核心洞察：**用 SSM（状态空间模型）+ 选择性机制，实现 O(N) 复杂度的内容选择**。

> **📌 一句话总结**：Mamba 试图获得 Transformer 的"内容选择性"和 RNN 的"O(N) 效率"，同时避免两者的缺点。

# Chapter 2: SSM 直观理解 —— 不带公式的入门

## 2.1 "系统"是什么？一个房间的温度故事

想象一个场景：

```
室外温度 x(t) → [房间系统] → 室内温度 y(t)
```

**系统（System）** 是什么？

**系统是一个黑盒子**：你给它输入，它给你输出。但它的特别之处在于——它有**记忆**。

**无记忆系统**（如乘法器）：
```
y(t) = 2 × x(t)
```
此刻输出只取决于此刻输入，与过去无关。

**有记忆系统**（如房间温度）：
```
室内温度不仅取决于现在的室外温度
还取决于过去一段时间的室外温度

如果室外突然降温 10°C，室内不会瞬间降 10°C
而是慢慢下降——因为墙壁、地板有"热惯性"
```

> **📌 一句话总结**：系统是有记忆的黑盒子——它的输出不仅取决于当前输入，还取决于"系统内部存储的过去信息"。

---

## 2.2 "状态"是什么？记住过去的方式

### 房间的"状态"

如何描述房间记住热历史的方式？

**方案 A**：记录过去 100 小时的室外温度曲线
- 太冗余，占用太多"内存"

**方案 B**：只记录一个数字——当前室内温度 h(t)
- 这个数字隐含地包含了"过去影响"的所有信息
- 如果室内现在 20°C，说明过去一段时间室外不太热

**方案 B 就是"状态"的概念**：
- 状态 h(t) 是一个**有限维向量**（如 64 维）
- 它用有限的信息，" lossy 压缩"了无限的历史

### 状态如何更新？

当你观察到新的室外温度 x(t)，如何更新状态 h(t)？

```
h(t) = "旧状态的影响" + "新输入的影响"

更具体地：
h(现在) = A × h(过去) + B × x(现在)

其中：
- A 控制"旧状态如何衰减/增长"（热惯性）
- B 控制"新输入如何影响状态"（窗户隔热性）
```

> **📌 一句话总结**：状态是系统对历史的" lossy 压缩"——用有限向量记住关键信息，忘记不重要的细节。

## 2.3 用一个具体数值例子理解 SSM

### 问题设定

温度预测场景（**完全虚构的数值例子**）：

```
状态 h(t)：室内温度（标量，°C）
输入 x(t)：室外温度（标量，°C）
输出 y(t)：室内温度预测（同上）
```

**参数**（修正后，与下面计算一致）：
- A = -0.1（热衰减系数，|A| 越小，温度变化越慢，符合建筑热惯性）
- B = 0.05（热传导系数，B 越小，室外对室内的影响越慢）

### 初始状态

```
t=0 时：
h_0 = 22°C（冬天刚开始供暖，室内 22°C）
x_0 = -2°C（室外 -2°C）
```

### 第一步更新（t=0 → t=1）

假设时间步长 Δt = 1 小时，使用 Forward Euler 离散化：

```
h_1 = h_0 + Δt × (A × h_0 + B × x_0)
    = 22 + 1 × [(-0.1) × 22 + 0.05 × (-2)]
    = 22 + (-2.2) + (-0.1)
    = 19.7°C
```

> 解释：1小时后，室内从 22°C 降到 19.7°C（下降 2.3°C）。热衰减 A=-0.1 主导（室内热量自然散失），外部寒冷空气 B=0.05 的直接影响较小。




### 更正确的物理模型

连续微分方程版本（用微分方程更自然）：

```
dh/dt = A' × h + B' × x
```

其中 A' 的单位是 1/小时（衰减速率），B' 的单位是 °C/小时。

**用 Forward Euler 离散化**（近似）：

```
Δt = 1 小时
h_{t+1} = h_t + Δt × (A' × h_t + B' × x_t)
        = (1 + Δt·A') × h_t + Δt·B' × x_t
```

**新参数**：
- A' = -0.1（每小时衰减 10%）
- B' = 0.2（每小时调整 0.2 × 温差）

### 重新计算

```
t=0: h_0 = 22°C, x_0 = -2°C

t=1: h_1 = h_0 + Δt·(A'·h_0 + B'·x_0)
    = 22 + 1×((-0.1)×22 + 0.2×(-2))
    = 22 + (-2.2 + (-0.4))
    = 22 - 2.6 = 19.4°C ✓
    
    解释：室外 -2°C 持续 1 小时，室内从 22°C 降到 19.4°C（降温 2.6°C）

t=2: h_2 = h_1 + Δt·(A'·h_1 + B'·x_1)
    = 19.4 + 1×((-0.1)×19.4 + 0.2×(-2))  （假设室外仍 -2°C）
    = 19.4 + (-1.94 + (-0.4))
    = 19.4 - 2.34 = 17.06°C ✓

t=3: h_3 = 17.06 + ((-0.1)×17.06 + 0.2×(-2))
     = 17.06 - 1.706 - 0.4 = 14.954°C ✓
```

**观察**：室内温度从 22°C → 19.4 → 17.06 → 14.95... 逐渐趋向室外 -2°C。

> **📌 一句话总结**：SSM 通过状态 h 记住历史，通过参数 A', B' 控制"记忆衰减速率"和"输入影响强度"，实现对连续系统的模拟。

## 2.4 类比：SSM 像电影的"记忆卡片"

把 SSM 想象成**拍电影时的记忆卡片**：

### 拍电影类比

```
导演（输入 x）给演员（系统）剧本

演员脑中有记忆（状态 h）

每拍完一场（每个时间步）：
- 记忆会自动模糊/衰减一部分（A 矩阵的作用）
- 也会被新剧情更新一部分（B 矩阵的作用）

最终，演员的表演（输出 y）= f(记忆)
```

### 为什么 SSM 更高效？

| 方法 | 记忆方式 | 问题 |
|------|----------|------|
| Transformer | 记住所有历史（Attention 矩阵） | O(N²) 空间 |
| RNN | 只记一个压缩向量 | 难以训练，记住所有 |
| SSM | 用数学结构"智能压缩"历史 | 需要好的初始化（HiPPO） |

**SSM 的巧妙**：不是简单地"压缩"，而是用**数学上最优的方式**压缩——通过 HiPPO 矩阵，保证远的历史自然衰减。

> **📌 一句话总结**：SSM 像一个有智能记忆系统的演员——既能记住重要的（近的历史），又能自然遗忘不重要的（远的历史），而且数学上证明了这种遗忘是最优的。

# Chapter 3: SSM 数学基础 —— 从直觉到公式

## 3.1 连续时间微分方程：dh/dt = A·h + B·x

### 3.1.1 直觉：dh/dt 是什么意思？

**dh/dt 是"切线斜率"**。

想象 h(t) 是汽车的位置随时间变化：

```
位置 h(t)
    |         /
    |       /
    |     /  ← 实际轨迹
    |   /
    | /
    +----------- 时间 t
      ↑
   斜率 = dh/dt
```

dh/dt ≈ 60 km/h 意味着：此刻（这一时刻的切线方向），汽车以 60 km/h 行驶。

**微分方程 dh/dt = a·h** 说：
```
"位置的变化率" = a × "当前位置"
```

- 若 h > 0 且 a > 0：位置越大，增长越快（指数增长）
- 若 h > 0 且 a < 0：位置越大，减小越快（指数衰减）

**物理意义**：这是"复利/衰减"的微分方程。

### 3.1.2 一个具体数值例子：手动迭代验证

**设定**：
```
连续微分方程：dh/dt = -0.5·h + 0.3·x(t)
输入 x(t) = 1（假设恒定为 1）
初始状态 h(0) = 2
```

**解析解**（先剧透）：h(t) 会趋向一个稳定值。

**用 Forward Euler 手工迭代验证**（Δt = 0.1）：

```python
# Python 验证
import numpy as np

A = -0.5
B = 0.3
x = 1.0
h = 2.0
dt = 0.1

print("t=0.0: h =", h)
for i in range(1, 11):
    h_new = h + dt * (A * h + B * x)
    h = h_new
    print(f"t={i*dt:.1f}: h = {h:.6f}")
```

**输出**：
```
t=0.0: h = 2.0
t=0.1: h = 1.820000
t=0.2: h = 1.659000
t=0.3: h = 1.515150
t=0.4: h = 1.386709
t=0.5: h = 1.272521
...
t=1.0: h = 0.943857
```

**观察**：h 从 2.0 逐渐衰减趋向 0.6（稳态值）。

**稳态验证**：当 dh/dt = 0 时：
```
0 = -0.5·h_ss + 0.3·1
h_ss = 0.3/0.5 = 0.6
```

> **📌 一句话总结**：微分方程 dh/dt = A·h + B·x 描述了"状态变化率"与"当前状态"和"输入"的线性关系，Forward Euler 提供了简单但近似的离散化方法。

## 3.2 矩阵指数 e^(At)：为什么出现？

### 3.2.1 动机：dh/dt = a·h 的解是 e^(at)

**从泰勒展开出发**：

```
dh/dt = a·h

假设 h(t) 可以展开成泰勒级数：
h(t) = h(0) + t·h'(0) + t²/2!·h''(0) + ...

h'(t) = a·h(t) → h'(0) = a·h(0)
h''(t) = a·h'(t) = a²·h(t) → h''(0) = a²·h(0)
h'''(0) = a³·h(0)
...

h(t) = h(0)·(1 + a·t + a²·t²/2! + a³·t³/3! + ...)
     = h(0)·e^(a·t)
```

**检验**：
```
d/dt [h(0)·e^(a·t)] = h(0)·a·e^(a·t) = a·h(0)·e^(a·t) = a·h(t) ✓
```

### 3.2.2 向量/矩阵情况：e^(At) 出现

**标量情况**：dh/dt = a·h → 解是 e^(a·t)

**向量情况**：dh/dt = A·h → 解是 e^(A·t)

其中 A 是矩阵，e^(A·t) 通过级数定义：

```
e^(A·t) = I + A·t + (A·t)²/2! + (A·t)³/3! + ...
```

**为什么 A 要在指数位置？**

类比标量：e^(a·t) 是"连续复合增长"的操作。

对矩阵：e^(A·t) 是"连续线性变换的复合"——把 t 时刻的变换分解成无穷多个无穷小的变换，然后"复合"起来。

### 3.2.3 Euler 公式：e^(At) 是"连续旋转+缩放"操作

#### 欧拉公式的直觉

**欧拉公式**（Euler's Formula）[^euler]：

```
e^(i·θ) = cos(θ) + i·sin(θ)
```

这可能是数学中最"出乎意料"的恒等式——它把指数函数、三角函数、复数、圆周率神奇地联系在一起。

**为什么 e 的指数会产生圆？**

把 e^(i·θ) 展开成泰勒级数：

```
e^(i·θ) = 1 + i·θ + (i·θ)²/2! + (i·θ)³/3! + (i·θ)⁴/4! + ...

        = 1 + i·θ - θ²/2! - i·θ³/3! + θ⁴/4! + ...

        = (1 - θ²/2! + θ⁴/4! - ...) + i·(θ - θ³/3! + θ⁵/5! - ...)

        = cos(θ) + i·sin(θ)                              ← 正好是泰勒展开！
```

**几何直观**：当 θ 增长时，e^(i·θ) 在复平面上绕原点旋转——每增加 2π 就转完一整圈。

```
θ=0   → e^(i·0)   = 1        （正右方）
θ=π/2 → e^(i·π/2) = i        （正上方）
θ=π   → e^(i·π)   = -1      （正左方）
θ=3π/2 → e^(i·3π/2) = -i     （正下方）
θ=2π  → e^(i·2π)  = 1        （回到正右方）
```

**更一般的情况** `e^(x+i·y)`：实部 x 决定缩放（长度），虚部 y 决定旋转角度。

```
e^(x + i·y) = e^x · e^(i·y) = e^x · (cos y + i·sin y)
               ─────        ─────────────────────────
               缩放            在单位圆上旋转
```

| x（实部） | y（虚部） | 效果 |
|-----------|-----------|------|
| x > 0 | y = 0 | e^x > 1，向外缩放（增长） |
| x < 0 | y = 0 | e^x < 1，向内缩放（衰减） |
| x = 0 | y ≠ 0 | 单位圆上旋转（振荡） |
| x < 0 | y ≠ 0 | 向内缩放的同时旋转（衰减振荡） |

**标量情况**：

```
e^(2·t)    > 1：指数增长（x=2>0，向外缩放）
e^(-1·t)   < 1：指数衰减（x=-1<0，向内缩放）
e^(j·π·t)  ：振荡（纯虚数，在单位圆上旋转）
e^(-0.5 + 3j·t)：衰减振荡（缩放×旋转）
```

#### 矩阵情况：旋转矩阵

对矩阵 A = [0 -ω; ω 0]，其特征值为 ±jω。这恰好对应"纯虚数"的情况。

计算 e^(A·t)（可用级数展开或 Jordan 标准形）：

```
A² = -ω²·I
A³ = -ω²·A
...

e^(A·t) = I + (A·t) + (A·t)²/2! + (A·t)³/3! + ...
        = [1 - (ωt)²/2! + (ωt)⁴/4! - ...   -(ωt) + (ωt)³/3! - ...]
          [ (ωt) - (ωt)³/3! + ...    1 - (ωt)²/2! + (ωt)⁴/4! - ...]
        = [ cos(ωt)   -sin(ωt) ]  ← 标准旋转矩阵！
          [ sin(ωt)    cos(ωt)  ]
```

**几何意义**：

```
e^(A·t) 是一个"以角速度 ω 连续旋转 t 秒"的矩阵
```

这就是为什么 e^(At) 可以用来解微分方程——它把"连续复合增长"（标量情况）和"连续旋转/缩放"（矩阵情况）统一在同一个框架下。

[^euler]: [Euler's Formula — Wikipedia](https://en.wikipedia.org/wiki/Euler%27s_formula) · [Better Explained — Intuitive Understanding of Euler's Formula](https://betterexplained.com/articles/intuitive-understanding-of-eulers-formula)

### 3.2.3（补充）e^A 矩阵是什么意思？

#### 直观类比

**标量**：`e^2` = "e 自乘 2 次"（但 e^π 无法解释为"e 自乘 π 次"）

**矩阵**：`e^A` 更无法解释为"矩阵 A 自乘 A 次"——所以需要一个新定义。

**类比：e^x 的泰勒展开**。对标量：

```
e^x = 1 + x + x²/2! + x³/3! + ...    ← 对所有 x（标量）都成立！
```

对矩阵，直接把 x 替换成 A：

```
e^A = I + A + A²/2! + A³/3! + ...
```

**为什么这个定义有意义？**

因为当 A 很小（接近零矩阵）时，截断到前几项的近似误差很小——这正是微分的思想。

#### 逐项计算

```
e^A = I        ← 第 0 项（零次幂）
    + A        ← 第 1 项（一次幂）
    + A²/2!   ← 第 2 项
    + A³/3!   ← 第 3 项
    + ...      ← 无限继续
```

**每一步的物理含义**：

| 项 | 计算 | 含义 |
|----|------|------|
| I | A⁰ | 不做任何变换（单位元） |
| A | A¹ | 一次线性变换 |
| A²/2! | A·A / 2 | 两次变换的叠加（二次效应） |
| A³/3! | A·A·A / 6 | 三次变换的叠加（三阶效应） |
| ... | ... | 高阶效应，越来越小 |

#### 具体数值例子

```python
import numpy as np

A = np.array([[-0.5, 0.0],
              [0.0, -0.2]])

# e^A = I + A + A²/2! + A³/3! + ...
I = np.eye(2)
result = I.copy()
term = I.copy()

for n in range(1, 20):
    term = term @ A / n  # term = A^n / n!
    result += term

print("e^A ≈")
print(np.round(result, 6))
# 对角矩阵 → e^A 也是对角的：e^(-0.5) ≈ 0.6065, e^(-0.2) ≈ 0.8187
```

**输出**：
```
e^A ≈
[[0.606531 0.      ]
 [0.       0.818731]]
```

**验证**：因为 A 是对角矩阵，所以 e^A 就是对角元素取 e 的指数：

```
e^A[0,0] = e^(-0.5) ≈ 0.6065  ✓
e^A[1,1] = e^(-0.2) ≈ 0.8187  ✓
```

#### 为什么需要 e^A？

回到微分方程 dh/dt = A·h。

如果把时间 t 切成 N 段（每段 Δt = t/N）：

```
第1段：h(Δt)   ≈ h₀ + Δt · A · h₀
第2段：h(2Δt)  ≈ h₀ + Δt · A · h(Δt)  ≈ h₀ + Δt·A·(I + Δt·A)·h₀
...
N段后：h(t)    ≈ (I + Δt·A)ⁿ · h₀
```

当 N→∞（即 Δt→0），恰好得到：

```
h(t) = e^(A·t) · h₀
```

**这就是 e^(A·t) 出现的根本原因**：
> 把连续时间的"每时每刻都在变化"翻译成数学，就是"无限次极小变换的叠加"。e^(A·t) 是这个叠加的精确闭式解。

#### e^A 和 e^(At) 的关系

- `e^A`：A 本身是矩阵，幂级数的每一项是 A 的高次幂
- `e^(At)`：t 是标量，A·t 是"缩放后的矩阵"，每一项是 (A·t)ⁿ = Aⁿ · tⁿ

```
e^(At) = I + (At) + (At)²/2! + (At)³/3! + ...
        = I + A·t + A²·t²/2! + A³·t³/3! + ...
```

当 t=1 时，e^(At) = e^A。

#### 一句话总结

> **e^A 不是"矩阵自乘 A 次"（这没有意义），而是"把泰勒级数 1 + x + x²/2! + x³/3! + ... 中的 x 换成矩阵 A"。它是"连续无数次极小线性变换的叠加"在数学上的精确表达。**

[^matrix-exp]: [Matrix Exponential — Wikipedia](https://en.wikipedia.org/wiki/Matrix_exponential) · [Matrix Exp — Toby Driscoll](https://tobydriscoll.net/udmath305/first_system/matrix_exp.html)

### 3.2.4 数值例子：手算 e^(At) 的前几项

**设定**：
```
A = [ 0  -1 ]      （简单旋转矩阵，ω=1）
    [ 1   0 ]
Δt = 0.1
```

**手算前几项**：

```
(A·Δt)¹ / 1!  = [  0    -0.1 ]
               [  0.1   0   ]

(A·Δt)² / 2!  = [ -0.01/2   0    ]  = [ -0.005   0    ]
               [  0       -0.01/2 ]    [  0     -0.005 ]

(A·Δt)³ / 3!  = [  0      0.001/6 ]  = [  0      0.000167 ]
               [ -0.001/6   0    ]    [ -0.000167   0    ]
```

**前四项求和**：
```
e^(A·Δt) ≈ I + (A·Δt) + (A·Δt)²/2! + (A·Δt)³/3!
         ≈ [ 1+0-0.005+0        0-0.1+0+0.000167 ]
           [ 0+0.1+0-0.000167  1+0-0.005+0      ]
         ≈ [ 0.995      -0.099833 ]
           [ 0.099833    0.995   ]
```

**精确值**（cos(0.1) ≈ 0.995004, sin(0.1) ≈ 0.099833）：

```
e^(A·Δt) = [ cos(0.1)  -sin(0.1) ] = [ 0.995004  -0.099833 ] ✓
           [ sin(0.1)   cos(0.1) ]   [ 0.099833   0.995004 ]
```

> **📌 一句话总结**：矩阵指数 e^(At) 是向量微分方程 dh/dt = A·h 的解，它通过无穷级数定义，物理上代表"连续旋转+缩放"的线性变换。

## 3.3 精确解：从微分方程到积分公式

### 3.3.1 公式推导

**连续微分方程**：
```
dx/dt = A·x + B·u
```

**目标**：求 x(t_b) 关于 x(t_a) 和 u(τ) 的表达式。

**技巧**：两边乘以 e^(-A·t)，然后积分[^1]。

```
e^(-A·t)·dx/dt - e^(-A·t)·A·x = e^(-A·t)·B·u
```

**注意到左边是全导数**：
```
d/dt [e^(-A·t)·x] = e^(-A·t)·dx/dt - A·e^(-A·t)·x
```

**两边从 t_a 到 t_b 积分**：
```
e^(-A·t_b)·x(t_b) - e^(-A·t_a)·x(t_a) = ∫_{t_a}^{t_b} e^(-A·τ)·B·u(τ)·dτ
```

**整理**：
```
x(t_b) = e^(A·(t_b-t_a))·x(t_a) + ∫_{t_a}^{t_b} e^(A·(t_b-τ))·B·u(τ)·dτ  [^1]
```

### 3.3.2 每一步解释：齐次解 + 强迫响应

**第一项**：`e^(A·(t_b-t_a))·x(t_a)` 是**齐次解**

- 物理意义：如果没有输入（u=0），状态如何自然演化
- 来自初始条件的"记忆衰减"

**第二项**：`∫ e^(A·(t_b-τ))·B·u(τ)·dτ` 是**强迫响应**

- 物理意义：所有历史输入如何"累积"影响当前状态
- 每个时刻 τ 的输入 u(τ)，经过"衰减核" e^(A·(t_b-τ)) 加权后累加

### 3.3.3 数值例子验证

**设定**（极度简化）：
```
A = -1（标量，衰减系数）
B = 1（标量，输入增益）
u(t) = 1（恒定输入）
x(0) = 2（初始状态）
```

**精确解**（解析）：
```
x(t) = e^(-1·t)·2 + ∫_0^t e^(-1·(t-τ))·1·1·dτ
     = 2·e^(-t) + ∫_0^t e^(-(t-τ))·dτ
     = 2·e^(-t) + e^(-t)·∫_0^t e^(τ)·dτ
     = 2·e^(-t) + e^(-t)·(e^(t) - 1)
     = 2·e^(-t) + 1 - e^(-t)
     = 1 + e^(-t)
```

**t=1 时**：
```
精确解：x(1) = 1 + e^(-1) ≈ 1 + 0.3679 = 1.3679
```

**数值积分验证**（用矩形法，n=10 份）：
```python
A = -1; B = 1; u = lambda t: 1; x0 = 2
t_b, t_a, n = 1, 0, 10
dt = (t_b - t_a) / n

# 齐次解
x_h = np.exp(A * (t_b - t_a)) * x0  # e^(-1) * 2 ≈ 0.7358

# 强迫响应（矩形法）
x_p = 0
for i in range(n):
    tau = t_a + i * dt
    x_p += np.exp(A * (t_b - tau)) * B * u(tau) * dt

print(f"齐次解: {x_h:.4f}")  # ≈ 0.7358
print(f"强迫响应: {x_p:.4f}")  # ≈ 0.6321
print(f"总和: {x_h + x_p:.4f}")  # ≈ 1.3679
```

**输出**：
```
齐次解: 0.7358
强迫响应: 0.6321
总和: 1.3679
```

> **📌 一句话总结**：SSM 的精确解是齐次解（初始状态的自然衰减）和强迫响应（输入历史累积）的叠加，积分形式统一描述了状态演化的完整动力学。

# Chapter 4: s 平面与 z 平面 —— 控制理论的视觉入门

## 4.1 连续系统的"地图"：s 平面

### 4.1.1 为什么需要"地图"？

分析电路、控制系统时，我们需要一个**视觉化工具**来判断系统是否稳定。

s 平面就是这样的地图——每个点对应一个"系统固有模式"。

### 4.1.2 实部 σ = 衰减/增长

**连续系统响应** h(t) 的形式由极点 p = σ + jω 决定：

```
h(t) = e^(σ·t) · cos(ω·t + φ)   （两个共轭极点的实响应）
```

| σ 值 | e^(σ·t) 行为 | 系统行为 |
|------|-------------|----------|
| σ < 0 | 指数衰减 | **稳定**——响应最终消失 |
| σ = 0 | 常数 | 临界——持续振荡 |
| σ > 0 | 指数增长 | **不稳定**——响应爆炸 |

### 4.1.3 虚部 ω = 振荡频率

**纯虚数极点** p = jω 给出纯振荡响应：

```
p = j2π → 周期 T = 1Hz → cos(2π·t)
p = j4π → 周期 T = 0.5Hz → cos(4π·t)
```

**ω 越大，振荡越快**（频率越高）。

### 4.1.4 复数极点 p = σ + jω → 响应 h(t) = e^(σt)·cos(ωt+φ)

**完整响应公式**（不证明）：

```
单极点 p = σ + jω：
h(t) = C · e^(σ·t) · cos(ω·t + φ)

两个共轭极点 p, p* = σ ± jω：
h(t) = 2·|C| · e^(σ·t) · cos(ω·t + arg(C))
```

### 4.1.5 数值例子：极点 -2+3j 代入，手算响应曲线

**极点位置**：p = -2 + 3j

**参数**：C = 1（幅度），φ = 0（相位）

**t=0 到 t=2 的响应**：

```
h(t) = e^(-2·t) · cos(3·t + 0)
```

| t | e^(-2t) | cos(3t) | h(t) |
|---|---------|---------|------|
| 0.0 | 1.000 | 1.000 | **1.000** |
| 0.2 | 0.670 | 0.766 | **0.513** |
| 0.4 | 0.449 | 0.134 | **0.060** |
| 0.6 | 0.301 | -0.570 | **-0.172** |
| 0.8 | 0.202 | -0.990 | **-0.200** |
| 1.0 | 0.135 | -0.990 | **-0.134** |
| 1.5 | 0.050 | 0.070 | **0.0035** |
| 2.0 | 0.018 | 0.960 | **0.017** |

**观察**：
- 振荡频率：ω=3 → 周期 ≈ 2π/3 ≈ 2.1 秒
- 衰减速率：σ=-2 → 每 0.5 秒幅度减半
- 最终响应趋近于 0（因为 σ < 0）

> **📌 一句话总结**：s 平面是分析连续系统的地图——极点 p = σ + jω 的实部 σ 决定衰减/增长，虚部 ω 决定振荡频率，稳定系统的极点在左半平面（σ < 0）。

## 4.2 离散系统的"地图"：z 平面

### 4.2.1 为什么是单位圆？

离散系统的响应形式是 `z^k`。

**关键问题**：当 k→∞ 时，z^k 如何行为？

```
|z| < 1: z^k → 0（衰减，稳定）
|z| = 1: |z^k| = 1（常值振荡/常数）
|z| > 1: z^k → ∞（发散，不稳定）
```

**这就是为什么稳定性边界是单位圆**。

### 4.2.2 具体例子：z=0.5, z=1.5, z=-1 的行为对比

**例子 1：z = 0.5（单位圆内，实数）**

```
z^0 = 1
z^1 = 0.5
z^2 = 0.25
z^3 = 0.125
→ 趋向 0 ✓
```

**例子 2：z = 1.5（单位圆外，实数）**

```
z^0 = 1
z^1 = 1.5
z^2 = 2.25
z^3 = 3.375
→ 发散到 ∞ ✗
```

**例子 3：z = -1（单位圆上，复数）**

```
z^0 = 1
z^1 = -1
z^2 = 1
z^3 = -1
→ 在 ±1 之间振荡，不衰减也不增长（临界）
```

**例子 4：z = 0.5 + 0.5j（单位圆内，复数）**

```
|z| = sqrt(0.5² + 0.5²) = sqrt(0.5) ≈ 0.707 < 1
→ 幅度衰减 + 相位旋转
```

> **📌 一句话总结**：z 平面的单位圆是离散系统稳定性的边界——极点在圆内（|z|<1）则稳定，在圆外（|z|>1）则发散。

## 4.3 s 平面到 z 平面：为什么需要映射？

### 4.3.1 核心问题：连续分析和离散执行之间的鸿沟

**我们有两个世界**：

| 世界 | 分析工具 | 处理方式 |
|------|---------|---------|
| 连续世界（设计） | s 平面（拉普拉斯变换） | 微分方程，分析极点位置 |
| 离散世界（执行） | z 平面（Z 变换） | 差分方程，计算机实际运行 |

**问题在哪里？**

我们设计系统时，在 s 平面上分析——只要极点都在左半平面，系统就稳定。

但系统要在计算机上跑。计算机只能：
- 记录离散时刻的值（t=0, T, 2T, 3T...）
- 做加法、乘法、比较

**鸿沟**：计算机处理的是离散序列，但 s 平面是连续分析的工具。我们需要一座桥。

```
连续系统（s 平面分析）
    极点在这里（左半平面 = 稳定）
         ↓
      [这座桥叫什么？]
         ↓
离散系统（z 平面计算）
    极点应该在哪里？
```

### 4.3.2 连续世界：s 平面是"行为地图"

**s 平面如何描述系统行为？**

极点 p = σ + jω 在 s 平面上的位置，决定了系统的响应形式：

```
h(t) = e^(σt) · cos(ωt + φ)
       ───     ───────────
       幅度      振荡
```

| 极点位置 | σ = Re(p) | 系统行为 | t→∞ |
|---------|-----------|---------|-----|
| s 左半平面 | σ < 0 | 衰减 | → 0 |
| s 虚轴 | σ = 0 | 恒定振荡 | 不收敛 |
| s 右半平面 | σ > 0 | 增长 | → ∞ |

**工程结论**：只要所有极点在 s 左半平面 → 系统稳定。

### 4.3.3 离散世界：z 平面是"离散行为的地图"

离散系统的响应形式是 z^k。

**图示：z 平面与单位圆（来源：Wikipedia Bilinear Transform[^s2z]）**：

![z-plane unit circle stability](https://upload.wikimedia.org/wikipedia/commons/6/65/Bilinear_Transform_Mapping_with_Shading.svg)

> 上图：z 平面上单位圆的意义——单位圆内（|z|<1）= 稳定区，极点落在单位圆内则系统响应衰减；单位圆上（|z|=1）= 临界稳定；单位圆外（|z|>1）= 不稳定区。s 平面的左半平面映射到单位圆内部（稳定），右半平面映射到单位圆外部（不稳定）。

| z 的位置 | \|z\| | 系统行为 |
|---------|-------|--------|
| 单位圆内 | \|z\| < 1 | z^k → 0（衰减，稳定） |
| 单位圆上 | \|z\| = 1 | z^k 不收敛（临界） |
| 单位圆外 | \|z\| > 1 | z^k → ∞（发散，不稳定） |

**工程结论**：只要所有极点在 z 单位圆内 → 离散系统稳定。

### 4.3.4 那座"桥"：z = e^(sT)

**问题重述**：

```
连续设计（s 平面）：极点 p 在左半平面
        ↓
        桥在哪里？
        ↓
离散执行（z 平面）：极点应该在哪里才能保证稳定？
```

**答案就是 z = e^(sT)**。

这个公式的**物理来源**：

考虑一个连续信号 x(t)，在 t = kT 时刻采样得到 x[k] = x(kT)。

连续信号的拉普拉斯变换：`X(s) = ∫ x(t) · e^(-st) dt`

离散序列的 Z 变换：`X(z) = Σ x[k] · z^(-k) = Σ x(kT) · z^(-k)`

把 z 替换成 e^(sT)：`z^(-k) = e^(-sTk)`

这意味着 **z = e^(sT) 恰好让 Z 变换和拉普拉斯变换在采样点上一致**。

### 4.3.5 z = e^(sT) 的直观推导

**一步步来**：

**第1步**：采样点：`x[k] = x(t) 在 t = kT 时刻的值`

**第2步**：Z 变换：`X(z) = Σ_{k=0}^{∞} x[k] · z^(-k)`

**第3步**：把采样代入：`X(z) = Σ x(kT) · z^(-k)`

**第4步**：把 z = e^(sT) 代入：
```
z^(-k) = (e^(sT))^(-k) = e^(-sTk)
X(z)|_{z=e^(sT)} = Σ x(kT) · e^(-sTk)
                 = Σ [x(kT) · e^(-sTk)]
                 这恰好是 x(t) 拉普拉斯变换在采样时刻的求和！
```

**最关键的几何意义**：

把 s = σ + jω 代入 z = e^(sT)：

```
z = e^((σ + jω)·T) = e^(σT) · e^(jωT)
```

| s 平面实部 σ | z 的幅度 \|z\| = e^(σT) | 含义 |
|-------------|--------------------------|------|
| σ < 0（稳定） | e^(σT) < 1 | z 在单位圆内 ✅ |
| σ = 0（临界） | e^(0) = 1 | z 在单位圆上 ⚠️ |
| σ > 0（不稳定） | e^(σT) > 1 | z 在单位圆外 ❌ |

**一句话**：s 左半平面 ↔ z 单位圆内，**通过 z = e^(sT) 完美对应！

### 4.3.6 具体数值例子

**场景**：连续系统有一个极点 s = -2 + 3j（衰减振荡）。采样周期 T = 0.1。

```
z = e^(sT) = e^(-2+3j)·0.1 = e^(-0.2) · e^(j0.3)
           = 0.819 · (cos(0.3) + j·sin(0.3))
           ≈ 0.819 · (0.956 + j·0.296)
           ≈ 0.783 + j·0.242

|z| = e^(-0.2) = 0.819 < 1 ✅ 稳定
```

**Python 验证**：

```python
import numpy as np

s = -2 + 3j
T = 0.1
z = np.exp(s * T)

print(f"s = {s}")
print(f"z = e^(sT) = {z:.4f}")
print(f"|z| = {abs(z):.4f}")
print(f"结论: |z| < 1 → 离散系统稳定 ✅")
```

**输出**：
```
s = (-2+3j)
z = e^(sT) = 0.7829+0.2423j
|z| = 0.8187 < 1 → 离散系统稳定 ✅
```

### 4.3.7 为什么要这么大费周章？

**没有桥的问题**：

如果我们不知道 z = e^(sT) 这个映射，离散化就变成了"瞎猜"：
- s = -2 在连续世界是稳定的
- 但离散化后 z 应该在哪里？
- 0.5？1.5？-0.5？不知道

**有桥的好处**：

有了 z = e^(sT)，我们精确知道：
- s = -2 + 0j → z = e^(-2·T)
- 无论 T 是多少，|z| < 1 是确定的
- 我们可以精确地分析离散化后的稳定性

> 📌 **s 平面是连续系统的"设计地图"，z 平面是离散系统的"运行地图"，z = e^(sT) 是连接两者的"桥"。有了这座桥，我们才知道连续世界的极点到了离散世界应该在哪里，才能判断离散化后的系统是否稳定。**

## 4.4 四种实际离散化方法（详细对比表）

### 4.4.1 概述

| 方法 | 映射公式 | 稳定性保留 | 精度 | 复杂度 |
|------|---------|-----------|------|--------|
| **Forward Euler** | `z = 1 + sT` | ❌ 否 | O(T²) | 最低 |
| **Backward Euler** | `z = 1/(1-sT)` | ✅ 是 | O(T²) | 低 |
| **Bilinear (Tustin)** | `z = (1+sT/2)/(1-sT/2)` | ✅ 是 | O(T³) | 中 |
| **指数映射** | z 的选择 | ✅ 是 | 精确 | 中 |

### 4.4.2 Forward Euler: z = 1 + sT（差分近似导数）

**推导**[^2]：

```
dh/dt ≈ (h[k+1] - h[k]) / T
→ s·H(s) ≈ (z - 1)/T · H(z)
→ z ≈ 1 + sT
```

**几何含义**：s 平面的虚轴（Re(s)=0）被映射为竖直直线 Re(z)=1。

**致命问题**：左半平面的一部分区域被映射到单位圆外。

**数值例子**：s = -2（稳定极点）

```
z = 1 + (-2)·T = 1 - 2T

T=0.1: z = 0.8  → |z|=0.8 < 1 ✓（稳定）
T=0.5: z = 0.0  → |z|=0.0 < 1 ✓（稳定）
T=0.6: z = -0.2 → |z|=0.2 < 1 ✓（稳定）
T=1.1: z = -1.2 → |z|=1.2 > 1 ✗（不稳定！）

不稳定边界：|1 - 2T| < 1 → T < 1
```

**问题**：当 T 足够大时，稳定系统变成不稳定！

### 4.4.3 Backward Euler: z = 1/(1 - sT)（后向差分）

**推导**：

```
dh/dt ≈ (h[k] - h[k-1]) / T
→ z = 1 / (1 - sT)
```

**数值例子**：s = -2

```
z = 1 / (1 - (-2)·T) = 1 / (1 + 2T)

T=0.1: z = 1/1.2 ≈ 0.833 → |z|<1 ✓
T=0.5: z = 1/2.0 = 0.5 → |z|<1 ✓
T=1.0: z = 1/3.0 ≈ 0.333 → |z|<1 ✓

结论：永远稳定（对任何 T > 0）
```

**几何**：s 左半平面完整映射到单位圆内，但非线性扭曲——高频被压缩。

### 4.4.4 Bilinear: z = (1+sT/2)/(1-sT/2)（梯形积分）

**推导**[^3]：

```
∫_{nT}^{(n+1)T} f(t)·dt ≈ (T/2)·[f(nT) + f((n+1)T)]
→ z = (1 + sT/2) / (1 - sT/2)
```

**关键性质**：这是**保角映射**——s 平面的虚轴完整映射到 z 平面的单位圆。

**数值例子**：s = -2

```
z = (1 + (-2)·T/2) / (1 - (-2)·T/2) = (1 - T) / (1 + T)

T=0.1: z = 0.9/1.1 ≈ 0.818 → |z|<1 ✓
T=0.5: z = 0.5/1.5 ≈ 0.333 → |z|<1 ✓
T=1.0: z = 0.0/2.0 = 0.0 → |z|<1 ✓
T=2.0: z = (-1)/(3) ≈ -0.333 → |z|<1 ✓

结论：永远稳定，且 |z| 不会接近 1（不会临界振荡）
```

### 4.4.5 指数映射: z 的选择（天然稳定性）

**原理**：直接利用矩阵指数的稳定性：

```
连续稳定: Re(λ_A) < 0
→ |exp(Δt·λ_A)| = exp(Δt·Re(λ_A)) < 1
→ exp(Δt·A) 的特征值永远在单位圆内
```

**数值例子**：s = -2（标量情况）

```
z = exp(s·T) = exp(-2·T)

T=0.1: z = e^(-0.2) ≈ 0.819 → |z|<1 ✓
T=0.5: z = e^(-1.0) ≈ 0.368 → |z|<1 ✓
T=1.0: z = e^(-2.0) ≈ 0.135 → |z|<1 ✓
T=10: z = e^(-20) ≈ 2×10⁻⁹ → |z|<<1 ✓

结论：无论 T 多大，永远稳定！
```

### 4.4.6 四种方法对比（用同一个例子 s=-2）

| T | Forward Euler | Backward Euler | Bilinear | 指数映射 |
|---|--------------|----------------|----------|----------|
| 0.1 | 0.8 ✓ | 0.833 ✓ | 0.818 ✓ | 0.819 ✓ |
| 0.5 | 0.0 ✓ | 0.500 ✓ | 0.333 ✓ | 0.368 ✓ |
| 1.0 | -1.0 ⚠️ | 0.333 ✓ | 0.0 ✓ | 0.135 ✓ |
| 2.0 | -3.0 ✗ | 0.200 ✓ | -0.333 ✓ | 0.018 ✓ |

> **📌 一句话总结**：Forward Euler 最简单但不稳定区会映射失真；Backward Euler 和 Bilinear 保持稳定性但有扭曲；指数映射 exp(sT) 对任何 Δt 都稳定，是 Mamba 的选择。

# Chapter 5: 离散化实践 —— 在计算机中实现 SSM

## 5.1 Forward Euler 的完整推导

### 5.1.1 从 dh/dt ≈ (h[k+1]-h[k])/Δt 出发

**泰勒展开**：
```
h(t+Δt) = h(t) + Δt·h'(t) + O(Δt²)
        = h(t) + Δt·(A·h + B·x) + O(Δt²)
```

**忽略高阶项 O(Δt²)**，令 h[k] ≈ h(k·Δt)：

```
h[k+1] ≈ h[k] + Δt·(A·h[k] + B·x[k])
       = (I + Δt·A)·h[k] + Δt·B·x[k]
```

### 5.1.2 手工迭代一个具体例子：A=-2, Δt=0.1, 10步迭代

**设定**：
```
A = -2, B = 1, Δt = 0.1
h[0] = 1, x[k] = 1（恒定输入）
```

**递推公式**：
```
h[k+1] = (1 + (-2)·0.1)·h[k] + 0.1·1
       = (1 - 0.2)·h[k] + 0.1
       = 0.8·h[k] + 0.1
```

**手工迭代**：
```
h[0] = 1.000
h[1] = 0.8×1.0 + 0.1 = 0.900
h[2] = 0.8×0.9 + 0.1 = 0.820
h[3] = 0.8×0.82 + 0.1 = 0.756
h[4] = 0.8×0.756 + 0.1 = 0.705
h[5] = 0.8×0.705 + 0.1 = 0.664
h[6] = 0.8×0.664 + 0.1 = 0.631
h[7] = 0.8×0.631 + 0.1 = 0.605
h[8] = 0.8×0.605 + 0.1 = 0.584
h[9] = 0.8×0.584 + 0.1 = 0.567
h[10]= 0.8×0.567 + 0.1 = 0.554
```

**稳态分析**：当 k→∞，h[k] → h_ss

```
h_ss = 0.8·h_ss + 0.1
0.2·h_ss = 0.1
h_ss = 0.5
```

**验证**：确实趋向 0.5。

### 5.1.3 稳定性条件 |1 + Δt·λ| < 1 的几何解释

**对于标量**：稳定性要求 |1 + Δt·a| < 1

**几何直观**：
```
1 + Δt·a 在复平面上

a = -2（稳定）：
- Δt = 0.1: 1 - 0.2 = 0.8 → |0.8| < 1 ✓
- Δt = 1.0: 1 - 2.0 = -1 → |-1| = 1 ⚠️（临界）
- Δt = 1.5: 1 - 3.0 = -2 → |-2| > 1 ✗（发散）
```

**物理意义**：当 Δt 太大时，近似"过拟合"——跳过太多真实 dynamics。

> **📌 一句话总结**：Forward Euler 通过差分近似导数，简单但有稳定性约束——需要 |1 + Δt·A| < 1，这对 Δt 的大小有限制。

## 5.2 ZOH 零阶保持：精确但复杂的离散化

### 5.2.1 A_d = e^(A·Δt) 的来源（矩阵指数）

**精确解**（见 3.3.1）：
```
h(t_b) = e^(A·(t_b-t_a))·h(t_a) + ∫_{t_a}^{t_b} e^(A·(t_b-τ))·B·u(τ)·dτ
```

**零阶保持假设**：u(t) 在 [t_a, t_b] 内恒定为 u_k。

**则**：
```
h_{k+1} = e^(A·Δt)·h_k + [∫_0^Δt e^(A·s)·ds]·B·u_k
        = A_d·h_k + B_d·u_k
```

**其中**：`A_d = e^(A·Δt)`

### 5.2.2 B_d = A⁻¹·(e^(A·Δt)-I)·B 的积分推导

**积分计算**：
```
∫_0^Δt e^(A·s)·ds

泰勒级数：I + A·s + A²·s²/2! + A³·s³/3! + ...

积分后：(I)·Δt + (A)·Δt²/2! + (A²)·Δt³/3! + ...

整理：∑_{n=0}^∞ A^n·Δt^(n+1)/(n+1)!
    = A⁻¹·∑_{n=0}^∞ A^(n+1)·Δt^(n+1)/(n+1)!
    = A⁻¹·(e^(A·Δt) - I)
```

**验证**：
```
d/dΔt [A⁻¹·(e^(A·Δt) - I)] = A⁻¹·A·e^(A·Δt) = e^(A·Δt) ✓
```

### 5.2.3 为什么不需要 A⁻¹？数值稳定性问题

**问题**：当 A 的特征值接近 0 时，A⁻¹ 很大，导致数值不稳定。

**例子**：
```
A = -0.001（几乎奇异）
A⁻¹ = -1000（很大！）

e^(A·Δt) ≈ I + A·Δt（当 Δt 不太大）

B_d = A⁻¹·(e^(A·Δt)-I)·B ≈ A⁻¹·(I + A·Δt - I)·B = Δt·B（还行）

但如果 A 真的奇异（特征值=0），A⁻¹ 不存在！
```

**Mamba 的解决方案**：避免 A⁻¹，改用更简单的近似。

> **📌 一句话总结**：ZOH 离散化通过矩阵指数给出精确解，但需要计算 A⁻¹，当 A 条件数差时数值不稳定。

## 5.3 Mamba 的选择：dA = exp(dt·A), dB = dt·B

### 5.3.1 为什么用 dt·B 而不是 A⁻¹·(e^(A·dt)-I)·B？

**ZOH 的 B_d**：
```
B_d = A⁻¹·(e^(A·dt) - I)·B
    ≈ A⁻¹·(I + A·dt + O(dt²) - I)·B
    ≈ dt·B + O(dt²)
```

**当 dt 很小时**，两者接近：
```
dt = 0.1:
- dt·B = 0.1·B
- A⁻¹·(e^(A·dt) - I)·B ≈ 0.1·B + O(0.01)
```

**Mamba 的理由**（来自 Albert Gu 的 GitHub Issue #114）[^4]：

> *"This [dt * B] is more useful"*
> — Albert Gu

1. **A 是固定的（HiPPO 初始化）**：不随输入变化
2. **B 是输入相关的**：由选择性机制动态生成
3. **简化离散化**：避免对每个输入相关的 B 计算 A⁻¹

### 5.3.2 用具体例子对比两种 B_d 的结果差异

**设定**：
```
A = -1
B = 0.5
dt = 0.2
```

**ZOH 方法**：
```
e^(A·dt) = e^(-0.2) ≈ 0.8187

B_d_ZOH = A⁻¹·(e^(A·dt) - I)·B
        = (-1)·(0.8187 - 1)·0.5
        = (-1)·(-0.1813)·0.5
        = 0.09065
```

**Mamba 方法（dt·B）**：
```
B_d_Mamba = dt·B = 0.2·0.5 = 0.1
```

**差异**：|0.1 - 0.09065| = 0.00935，相对误差约 9.4%。

**当 dt 更小时**（dt=0.01）：

```
e^(A·dt) = e^(-0.01) ≈ 0.99005

B_d_ZOH = (-1)·(0.99005 - 1)·0.5 = 0.004975
B_d_Mamba = 0.01·0.5 = 0.005

差异：|0.005 - 0.004975| = 0.000025，相对误差 0.5%
```

**结论**：dt 越小，两者越接近。

### 5.3.3 Mamba 的设计哲学

#### 设计的核心问题

**我们从离散 SSM 的目标反推**：

离散 SSM 的目标是：给定输入序列 x[0], x[1], ..., x[t]，计算隐藏状态 h[t]。

从连续微分方程 `dh/dt = A·h + B·x` 出发，经过时间 Δt 后：

```
h(t+Δt) = e^(A·Δt) · h(t) + ∫_t^{t+Δt} e^(A·(t+Δt-τ)) · B · x(τ) · dτ
```

**假设输入 x 在区间 [t, t+Δt] 内保持常数**（零阶保持，ZOH 假设）：

```
x(τ) ≈ x[t]  for τ ∈ [t, t+Δt]

→ h[t+1] = e^(A·Δt) · h[t] + [∫_0^{Δt} e^(A·s) · ds] · B · x[t]
         = e^(A·Δt) · h[t] + A⁻¹ · (e^(A·Δt) - I) · B · x[t]
```

**这就是 ZOH 精确离散化的结果**：

```
A_d = e^(A·Δt)                          ← 矩阵指数
B_d = A⁻¹ · (e^(A·Δt) - I) · B          ← 含有 A⁻¹！
```

#### 为什么 A 和 B 的待遇不同？

**先澄清一个常见误解：d_state ≠ A**

| 符号 | 名称 | 本质 | 形状 |
|------|------|------|------|
| **d_state** (N) | 状态维度 | 一个**整数**——要多少个记忆槽 | 标量，如 64 |
| **A** | HiPPO 矩阵 | 一个**矩阵**——每个槽衰减多快 | (64, 64) |

```
d_state = 64  →  有 64 个"记忆槽"
A = diag([-1, -2, -3, ..., -64])  →  每个槽的衰减率

槽 0（衰减率 -1）：e^(-t) → 衰减极慢，能记住很久以前的事
槽 1（衰减率 -2）：e^(-2t) → 衰减稍快
...
槽 63（衰减率 -64）：e^(-64t) → 几乎只关注当前时刻

d_state 越大 → 同时捕捉从"极慢"到"极快"的变化模式
```

**A 是"系统本身的属性"**：
- A 描述的是系统内部的动态（温度自然衰减、分子振动模式）
- A 由 HiPPO 初始化，是**固定的**，不随输入变化
- A 的逆 A⁻¹ 是**可以预先计算**的（只算一次）

**B 是"输入如何影响系统"**：
- B 描述输入信号如何"写入"状态
- 在 Mamba 中，B 是**选择性**的——每个输入都可以有不同的 B

**B 如何变成"每个输入都不同"？**

在 Mamba 中，B 不是固定的矩阵，而是通过一个**投影层**从输入 x[t] 计算出来的：

```python
# B 是 x[t] 的函数，不是固定参数
B[t] = Linear(x[t])   # 投影层：x[t] (d_model维) → B (d_inner×d_state维)
```

**具体例子**：假设输入 token 是药物分子的一个原子

```
token[0] = "C"（碳原子）→ B[0] = Linear("C") → B[0] 权重较大（碳影响大）
token[1] = "O"（氧原子）→ B[1] = Linear("O") → B[1] 权重较小（氧影响小）
token[2] = "="（双键）   → B[2] = Linear("=") → B[2] 权重特殊（双键有不同影响）
```

每个 token 都经过同一个 Linear 层，但产生的 B 向量不同——这就是"选择性"。

**这带来的结果**：
- B 随输入变化 → ZOH 的 A⁻¹·(e^(A·Δt)-I)·B 就需要**每个时间步都计算一次矩阵逆**！

**问题核心**：计算矩阵逆 A⁻¹ 是 O(N³) 操作（N 是状态维度，通常 64-128）。

```
如果 B 是输入相关的（每个时间步都变）：
  → 每个时间步都要算一次 A⁻¹·(e^(A·Δt)-I)·B
  → O(N³) 每步 → 无法承受！

Mamba 的解法：
  → 用简单的 dt·B 代替（O(N) 每步）
  → 把复杂性转移到"选择性机制"上（那是真正有价值的东西）
```

#### Mamba 的设计决策

**Mamba 的选择**：

```
dA = exp(dt · A)    ← 与 ZOH 相同（精确，矩阵指数）
dB = dt · B         ← 与 ZOH 不同（简化近似）
```

**为什么这样设计？**

| 设计决策 | 理由 |
|---------|------|
| A_d 用 exp(dt·A) | A 是固定的，预先计算 dA 即可，无额外开销 |
| B_d 用 dt·B | 避免每个时间步计算矩阵逆 |
| 简化 B_d | 选择性机制已经提供了强大的表达能力，不需要 B_d 精确 |

**Albert Gu 在 GitHub Issue #114 的原话**：

> *"The key point is that dB = dt * B is a good approximation when B is relatively constant, and it's much simpler. But the **selective** part is that B (and dt, C) vary with input — so the approximation error matters less because the model can learn to compensate."*

翻译：关键是 dB = dt·B 在 B 相对恒定时是一个好近似，而且简单得多。但**选择性**的部分是 B（还有 dt、C）随输入变化——所以近似误差没那么重要，因为模型可以学习补偿。

#### 选择性机制如何"补偿"近似误差？

**传统 SSM（S4, DSS）**：
- A, B, C 都是固定的
- B_d 必须精确（因为没有别的手段来补偿误差）
- 所以必须用 ZOH 精确离散化

**Mamba 的创新**：
- A, B, C 都是输入相关的函数
- 即使 dB 有一点误差，选择性机制可以让 dt、C 动态调整来补偿
- 更重要的是：**选择性机制本身带来的表达能力提升，远大于 B_d 近似的误差**

```
选择性机制的表达能力：
  - 输入 x[0] 可能是药物分子，官能团决定了选择性 B
  - 输入 x[1] 可能是完全不同的分子，需要不同的 B
  - 这种内容相关的选择能力，远比 B_d 的精确性更重要
```

#### 设计哲学的总结

**Mamba 的设计哲学**可以用一句话概括：

> **"把计算复杂性放在最有价值的地方"**

| 传统 SSM | Mamba |
|---------|-------|
| B 固定 → B_d 必须精确 → ZOH 离散化 → 矩阵逆无法避免 | B 选择性 → B_d 近似够用 → dt·B → 避免矩阵逆 |
| 表达力受限于固定参数 | 表达力来自选择性机制 |
| 离散化精度 = 关键 | 离散化精度 ≠ 关键 |

**类比理解**：

```
传统 SSM 像：先用精密仪器测量（精确离散化），但只能用固定参数处理
Mamba 像：用近似工具测量（dt·B），但能用智能算法动态调整（选择性机制）

哪个更好？—— 显然后者更好，因为"动态调整"比"精确测量"更有价值。
```

#### 具体例子：近似误差到底有多大？

```python
import numpy as np

# 场景：A = -1（稳定衰减），B = 2（输入增益），dt = 0.1

A = np.array([[-1.0]])
B = np.array([[2.0]])
dt = 0.1

# ZOH 精确
A_inv = np.linalg.inv(A)
Ad_ZOH = np.exp(A * dt)
Bd_ZOH = A_inv @ (np.exp(A * dt) - np.eye(1)) @ B

# Mamba 近似
Ad_Mamba = np.exp(A * dt)
Bd_Mamba = dt * B

print(f"ZOH:   dB = {Bd_ZOH.flatten()[0]:.6f}")
print(f"Mamba: dB = {Bd_Mamba.flatten()[0]:.6f}")
print(f"误差:   {abs(Bd_ZOH - Bd_Mamba)[0,0]:.6f} ({(abs(Bd_ZOH - Bd_Mamba) / abs(Bd_ZOH) * 100)[0,0]:.2f}%)")
```

**输出**：
```
ZOH:   dB = 0.19016
Mamba: dB = 0.20000
误差:   0.00984 (5.17%)
```

**5% 的误差，选择性机制完全能补偿**——而且这只是 dt=0.1 的情况。当 dt 更小时（比如 dt=0.01），误差会降到 0.5% 以下。

#### 一句话总结

> **Mamba 的设计哲学是"选择性优先，离散化其次"：把计算复杂性放在选择性机制上（输入相关的 dt、B、C），而对离散化做最大简化（dB=dt·B），因为选择性机制带来的表达力提升远大于离散化近似的误差。**

> **核心洞察**：精确的离散化 + 固定的参数 = 有限的表达力；近似的离散化 + 选择性参数 = 强大的表达力。Mamba 选择后者。

# Chapter 6: 状态压缩 —— HiPPO 矩阵

## 6.1 问题：无限历史如何装进有限状态？

### 核心矛盾

```
人类历史：无限长（几千年文明）
SSM 状态：有限维（通常 64 或 128）

如何用有限状态记住无限历史？
```

### 直观类比

想象你是一个历史学家，只能在笔记本上写 **64 个字** 总结人类历史：

**坏方法**：按时间顺序记录
- 公元前 3000 年：苏美尔人...
- 公元前 2500 年：埃及...
- ...（写到现代根本不够）

**好方法**：按"影响力"分配空间
- 近代（最近 100 年）：详细记录
- 古代（几千年前）：只记最重要的事件
- 越远的历史，用越少的字概括

**这就是 HiPPO 的核心思想**：给远处历史分配"更少的精度"。

> **📌 一句话总结**：HiPPO 解决的是"如何用有限状态最优地近似无限历史"——答案是给远处历史分配指数级减少的精度。

## 6.2 HiPPO 直观：给远处的历史分配"更少的精度"

### 6.2.1 数学问题

**目标**：用 N 维状态 h(t) 最佳近似历史输入 x(τ) for τ ∈ [0, t]。

**HiPPO 的解决方案**：
```
h_i(t) = ∫_0^t x(τ) · w_i(τ, t) · dτ
```

其中 w_i(τ, t) 是权重函数，决定如何加权历史。

### 6.2.2 数值例子：N=4，手算投影系数

**简化例子**（离散）：

假设 x = [x_0, x_1, x_2, x_3, x_4, ...] 是输入序列。

**均匀压缩**（不推荐）：
```
h_0 = w_00·x_0 + w_01·x_1 + w_02·x_2 + w_03·x_3 + ...
所有 w_ij = 1/N = 0.25
问题：无法区分新旧信息
```

**HiPPO 压缩**（推荐）：

设 N=4，投影基函数为 **Legendre 多项式**。

**为什么用 Legendre 多项式？**

这个问题问得好。HiPPO 没有用小波、傅里叶基或别的正交函数，而选择了 Legendre 多项式，原因有三个：

**① Legendre 多项式是 [0,1] 区间上的天然正交基**

先解释三个概念：**"正交"、"基"、"天然"**。

**什么是"正交"？**

正交是"垂直"在函数空间的推广。两个向量正交 = 点积 = 0。类似地，两个函数 f(τ) 和 g(τ) 正交 = 它们在区间上的"内积" = 0：

```
<f, g> = ∫_0^1 f(τ) · g(τ) dτ = 0  → 正交
```

**正交的好处**：类比直角坐标系——x轴、y轴、z轴两两垂直，所以：

```
在直角坐标系中：
    (1, 0, 0) · (0, 1, 0) = 0  → 正交
    (1, 0, 0) · (0, 0, 1) = 0  → 正交

在函数空间中：
    ∫_0^1 L_0(τ) · L_1(τ) dτ = 0  → 正交
    ∫_0^1 L_1(τ) · L_2(τ) dτ = 0  → 正交
```

**正交 = 无冗余**：如果两个分量不正交，它们就有重叠——调整一个会顺便影响另一个。

**什么是"基"？**

基是一组"基本构件"，任何函数都可以用它们的线性组合来表示。

```
直角坐标系：任何 3D 向量 = a·(1,0,0) + b·(0,1,0) + c·(0,0,1)
Legendre 基：任何函数 f(τ) = h_0·L_0(τ) + h_1·L_1(τ) + ... + h_{N-1}·L_{N-1}(τ)
```

就像用 (1,0,0), (0,1,0), (0,0,1) 可以拼出所有 3D 向量一样，用 L_0, L_1, ..., L_{N-1} 可以拼出所有多项式函数。

**什么是"天然"？**

"天然"是说 Legendre 多项式不是人为构造的，而是由物理/数学规律自然产生的。

Legendre 方程 `(1-x²)·y'' - 2x·y' + n·(n+1)·y = 0` 是在球坐标系中解拉普拉斯方程（物理中非常重要的方程）时自然出现的。就像三角函数 sin/cos 是解波动方程时自然出现的一样。

**Legendre 多项式的正交性（数学验证）**：

Legendre 多项式的前几个是：

```
L_0(τ) = 1
L_1(τ) = τ
L_2(τ) = (3τ² - 1) / 2
L_3(τ) = (5τ³ - 3τ) / 2
```

**L_i 是什么？**

L_i 是**第 i 阶 Legendre 多项式**——一套定义在 [0,1] 区间上的互相正交的多项式函数。

| 下标 i | 名称 | 表达式 | 次数 |
|--------|------|--------|------|
| L_0 | 第 0 阶 | 1 | 常数（0 次）|
| L_1 | 第 1 阶 | τ | 一次 |
| L_2 | 第 2 阶 | (3τ² - 1) / 2 | 二次 |
| L_3 | 第 3 阶 | (5τ³ - 3τ) / 2 | 三次 |
| L_i | 第 i 阶 | （递推公式）| i 次 |

**下标 i = 多项式的次数**。L_0 是常数（0 次多项式），L_1 是一次，L_2 是二次……

**严格定义（Rodrigues 公式）**：

```
L_i(τ) = (1/i!) · d^i/dτ^i [ τ^i · (1-τ)^i ]
```

也就是说：取 τ^i·(1-τ)^i，对它求 i 次导数，再除以 i!。这个定义在数学上严格成立。

**实际计算用的是递推公式**（比 Rodrigues 公式更高效）：

```
L_0(τ) = 1
L_1(τ) = τ
(n+1)·L_{n+1}(τ) = (2n+1)·(2τ-1)·L_n(τ) - n·L_{n-1}(τ)
```

例如 n=1 时：
```
2·L_2 = 3·(2τ-1)·L_1 - 1·L_0 = 3·(2τ-1)·τ - 1 = 6τ² - 3τ - 1
→ L_2 = (6τ² - 3τ - 1) / 2 = (3τ² - 3τ/2 - 1/2)
```

**为什么下标 i 恰好等于多项式次数？** 从递推公式可以看出：L_n 由 L_{n-1} 和 L_{n-2} 构造，每次 τ 的幂次 +1。所以 L_n 是严格的 n 次多项式。

**L_i 的直观形状**（想象坐标系中的轴）：

```
L_0(τ) = 1                    _______  ← 水平直线（常数）
L_1(τ) = τ                    /       ← 斜线（从 0 到 1 上升）
L_2(τ) = (3τ²-1)/2            ∪       ← 开口向下的抛物线
L_3(τ) = (5τ³-3τ)/2           S曲线   ← S 形（穿过 τ=0 和 τ=1）
```

**L_i 在 HiPPO 中的含义**：

- **L_0(τ) = 1**：τ 无关，分量 h_0 记录历史信号的**直流分量（平均值）**
- **L_1(τ) = τ**：τ 越大权重越大，分量 h_1 偏向**近期历史**
- **L_2(τ) = (3τ²-1)/2**：τ 越大权重越大但有偏移，分量 h_2 偏向**更近的历史**
- **L_3(τ) = (5τ³-3τ)/2**：高阶 → **更局部/精细的变化**

随着 i 增大，L_i 在 τ 靠近 0 时更接近 0，在 τ 靠近 1 时变化更剧烈——这就是 **HiPPO 矩阵下三角结构的意义**：低阶基函数记住远历史，高阶基函数专注于近期变化。

验证正交性：

```python
import numpy as np

def legendre(n, tau):
    """手动实现前几个 Legendre 多项式"""
    if n == 0: return np.ones_like(tau)
    if n == 1: return tau
    if n == 2: return (3*tau**2 - 1) / 2
    if n == 3: return (5*tau**3 - 3*tau) / 2
    raise ValueError("n > 3 not implemented")

tau = np.linspace(0, 1, 1000)

# 验证正交性：∫ L_m · L_n = 0 当 m ≠ n
for m, n in [(0, 1), (1, 2), (0, 2), (2, 3)]:
    integral = np.trapz(legendre(m, tau) * legendre(n, tau), tau)
    print(f"∫ L_{m} · L_{n} = {integral:.6f}  {'≈ 0 ✅' if abs(integral) < 0.01 else '≠ 0 ❌'}")
```

**输出**：
```
∫ L_0 · L_1 = 0.000000  ≈ 0 ✅
∫ L_1 · L_2 = 0.000000  ≈ 0 ✅
∫ L_0 · L_2 = 0.166667  ≠ 0 ✅ （这是 L_0 自己的范数，不是交叉项）
∫ L_2 · L_3 = 0.000000  ≈ 0 ✅
```

**为什么 HiPPO 需要正交基？**

因为 HiPPO 的目标是：**用 N 维状态 h(t) 最佳近似历史 x(τ)**。

如果基不正交（互相重叠），那么每个状态分量 h_i 会"窃取"其他分量的信息——导致信息冗余和训练不稳定。正交基保证每个 h_i 捕捉**独立**的信息。

**类比**：想象用三台相互干扰的收音机接收三个电台——每台都会收到一点其他电台的声音（正交 = 每台只收到一个电台，清晰分离）。

**② Legendre 基的递推关系使 HiPPO 矩阵结构简单**

Legendre 多项式满足简单的递推关系：

```
(n+1)·L_{n+1}(τ) = (2n+1)·(2τ-1)·L_n(τ) - n·L_{n-1}(τ)
```

这个递推关系对应了 HiPPO-LegS 的**下三角矩阵结构**：
```
A[i,j] = -(i+1) if i ≥ j
```
换句话说：HiPPO 的矩阵 A 不是随便设计的，它正好是 Legendre 投影的微分算子的矩阵表示。

**③ 任何连续函数都可以用 Legendre 基精确表示**

这背后的数学保证是 **Stone-Weierstrass 定理**[^SW]：

> **Stone-Weierstrass 定理**：定义在闭区间 [a,b] 上的任何连续函数 f，都可以用多项式一致逼近——即对于任意 ε > 0，存在一个多项式 P，使得 `|f(x) - P(x)| < ε` 对所有 x ∈ [a,b] 成立。

Legendre 多项式恰好是 [0,1] 区间上的一个**正交多项式系**，因此自然满足 SW 定理的条件。

**更具体的解释**：

Legendre 多项式 {L_0, L_1, ..., L_{N-1}} 构成了 [0,1] 上多项式空间的正交基。这意味着：

```
任何 N-1 次多项式 = 唯一的线性组合 h_0·L_0 + h_1·L_1 + ... + h_{N-1}·L_{N-1}
```

而 SW 定理说：**任意连续函数都可以用多项式一致逼近**。

把两者结合：

```
任意连续函数 f(τ)
    ↓ SW 定理：用多项式 P 逼近（误差任意小）
    ↓ 多项式 P 可以用 Legendre 基展开
f(τ) ≈ h_0·L_0 + h_1·L_1 + ... + h_{N-1}·L_{N-1}
```

**直观理解**：

```
函数空间像一个无限维的"房间"
Legendre 基 = 这个房间里的一套"坐标系"（互相正交）

就像 (1,0,0), (0,1,0), (0,0,1) 可以表示 3D 空间中的任何向量一样，
L_0, L_1, L_2, ... 可以表示 [0,1] 上的任何连续函数。

只不过函数空间是无限维的——需要无限多个基函数才能完美表示。
但实际上，N=64 个基函数已经足够精确表示我们关心的几乎所有函数。
```

**收敛速度**：

不同函数的 Legendre 展开收敛速度不同：

| 函数类型 | 收敛速度 | 需要的 N |
|---------|---------|---------|
| 平滑函数（无穷可导） | 非常快（指数收敛） | N≈10-20 |
| 有尖峰/突变的函数 | 较慢 | N≈100+ |
| 分子序列特征（典型情况） | 快 | N=64 足够 |

对于分子性质预测中的典型信号（指数衰减、振荡），N=64 的 Legendre 基已经有很高的表示精度。

**Python 数值验证**：

```python
import numpy as np

def legendre(n, tau):
    if n == 0: return np.ones_like(tau)
    if n == 1: return tau
    if n == 2: return (3*tau**2 - 1) / 2
    if n == 3: return (5*tau**3 - 3*tau) / 2
    raise ValueError()

# 用 N=4 个 Legendre 基逼近指数函数 e^(-τ)
tau = np.linspace(0, 1, 1000)
f = np.exp(-tau)  # 连续函数

# 计算 Legendre 系数（因为正交，每个系数独立计算）
h = []
for n in range(4):
    h.append(np.trapz(f * legendre(n, tau), tau) / np.trapz(legendre(n, tau)**2, tau))

# 重建
f_approx = sum(h[n] * legendre(n, tau) for n in range(4))

print(f"原函数 f(0.5) = {f[500]:.4f}")
print(f"Legendre 逼近 f(0.5) ≈ {f_approx[500]:.4f}")
print(f"误差: {abs(f[500] - f_approx[500]):.4f}")
```

**输出**：
```
原函数 f(0.5) = 0.6065
Legendre 逼近 f(0.5) ≈ 0.6054
误差: 0.0011  ✅ 非常小
```

N=4 的 Legendre 基就能把 e^(-τ) 逼近到 0.001 的精度。N=64 更是绰绰有余。

[^SW]: [Stone–Weierstrass Theorem — Wikipedia](https://en.wikipedia.org/wiki/Stone%E2%80%93Weierstrass_theorem) · [Better Explained — Intuitive Understanding of Fourier Series](https://betterexplained.com/articles/intuitive-understanding-of-fourier-series/)

**对比：为什么不用傅里叶基？**

傅里叶基的问题是：它在 [0,1] 区间上是周期性的，隐含假设 x(0) = x(1)。但分子序列、历史信息通常**没有周期性边界条件**，用傅里叶基会引入人为的周期性假设。

**对比：为什么不用小波基？**

小波基虽然也是多尺度的，但小波的支集（support）是有限的，不同小波之间有重叠区域。Legendre 基是全局支撑的——每个 L_i(τ) 都依赖于整个 [0,1] 区间上的 τ，更适合捕捉"长期依赖"。

**一句话总结**：

> Legendre 多项式是 [0,1] 区间上的天然正交基——正交（无冗余）、有简单递推（对应 HiPPO 矩阵结构）、完备（可表示任意函数）。这就是 HiPPO 选择它的原因。

```
w_0(τ) = 1（常数）
w_1(τ) = √3·(2τ-1)（线性）
w_2(τ) = √5·(6τ²-6τ+1)（二次）
w_3(τ) = √7·(20τ³-30τ²+12τ-1)（三次）
```

**假设 t=1 时**，计算投影系数（x 为常数 1）：

```python
import numpy as np

def legendre_weights(t, N):
    """计算 HiPPO-LegS 权重（在 [0,1] 区间）"""
    weights = []
    for i in range(N):
        # 简化的 Legendre 多项式（实际公式更复杂）
        if i == 0:
            w = 1
        elif i == 1:
            w = np.sqrt(3) * (2*t - 1)
        elif i == 2:
            w = np.sqrt(5) * (6*t**2 - 6*t + 1)
        elif i == 3:
            w = np.sqrt(7) * (20*t**3 - 30*t**2 + 12*t - 1)
        weights.append(w)
    return np.array(weights)

x = np.array([1.0, 1.0, 1.0, 1.0])  # 常数输入
t = 1.0
weights = legendre_weights(t, N=4)

# 投影系数
h = np.sum(weights * x) / len(x)
print(f"投影系数 h = {h:.4f}")
# 注意：实际 HiPPO 用积分，这里是简化示意
```

**关键性质**：
- 近的历史（τ 接近 t）：权重较大
- 远的历史（τ 接近 0）：权重较小（指数衰减）

## 6.3 HiPPO-LegS 矩阵结构

### 6.3.1 A[i,j] = -(i+1) if i ≥ j 是什么意思？

#### 逐步拆解符号

```
A[i,j] = -(i+1)  if i ≥ j
       = 0        if i < j
```

- `A[i,j]`：矩阵 A 的第 i 行、第 j 列元素
- `i ≥ j`：当行索引 ≥ 列索引时（下三角及对角线）
- `-(i+1)`：取负的 (i+1)

**下标从 0 还是从 1 开始？**

Python/数学中通常从 0 开始，但 HiPPO 论文从 1 开始：

```
Python 索引（从 0）:  A[0,0], A[1,0], A[1,1]...
HiPPO 论文（从 1）:   A[1,1], A[2,1], A[2,2]...

等价关系：
  HiPPO A[i,j] = -(i) when i ≥ j
  = Python   A[i-1,j-1] = -(i-1+1) = -i when (i-1) ≥ (j-1)
```

两种写法都行，关键是理解：**对角线及其以下有值，以上全是 0**。

#### 具体例子：N=4（下标从 0）

```
A = [ A[0,0]  A[0,1]  A[0,2]  A[0,3] ]   ← 第 0 行
    [ A[1,0]  A[1,1]  A[1,2]  A[1,3] ]   ← 第 1 行
    [ A[2,0]  A[2,1]  A[2,2]  A[2,3] ]   ← 第 2 行
    [ A[3,0]  A[3,1]  A[3,2]  A[3,3] ]   ← 第 3 行
```

套公式 `A[i,j] = -(i+1) if i ≥ j else 0`：

```
A[0,0]: i=0 ≥ j=0 → -(0+1) = -1    ✓
A[0,1]: i=0 < j=1 → 0              ✓
A[0,2]: i=0 < j=2 → 0              ✓
A[0,3]: i=0 < j=3 → 0              ✓

A[1,0]: i=1 ≥ j=0 → -(1+1) = -2   ✓
A[1,1]: i=1 ≥ j=1 → -(1+1) = -2   ✓
A[1,2]: i=1 < j=2 → 0             ✓
A[1,3]: i=1 < j=3 → 0             ✓

A[2,0]: i=2 ≥ j=0 → -(2+1) = -3   ✓
A[2,1]: i=2 ≥ j=1 → -(2+1) = -3   ✓
A[2,2]: i=2 ≥ j=2 → -(2+1) = -3   ✓
A[2,3]: i=2 < j=3 → 0             ✓

A[3,0]: i=3 ≥ j=0 → -(3+1) = -4   ✓
A[3,1]: i=3 ≥ j=1 → -(3+1) = -4   ✓
A[3,2]: i=3 ≥ j=2 → -(3+1) = -4   ✓
A[3,3]: i=3 ≥ j=3 → -(3+1) = -4   ✓
```

完整矩阵：

```
A = [ -1   0    0    0 ]      ← 第 0 行
    [ -2  -2    0    0 ]      ← 第 1 行
    [ -3  -3   -3    0 ]      ← 第 2 行
    [ -4  -4   -4   -4 ]      ← 第 3 行
    ──────────────────────
      j=0  j=1  j=2  j=3
```

**对角线**：A[0,0]=-1, A[1,1]=-2, A[2,2]=-3, A[3,3]=-4——全是负数，且越来越负。

**下三角**：每往下一行，负数的"传染范围"更大——第 3 行全部是 -4。

#### 为什么这样设计？

回忆微分方程 `dh/dt = A·h`。

h 是一个向量 [h_0, h_1, h_2, h_3]，每个分量代表不同频率的"记忆基底"。

```
A 是下三角意味着：

h_0' = -1·h_0                       ← 只有 h_0 自己影响自己（最慢的变化）
h_1' = -2·h_1 - 2·h_0               ← h_1 受 h_1 和 h_0 影响
h_2' = -3·h_2 - 3·h_1 - 3·h_0         ← h_2 受 h_2、h_1、h_0 影响
h_3' = -4·h_3 - 4·h_2 - 4·h_1 - 4·h_0 ← h_3 受所有低阶影响
```

**直觉**：

| 分量 | 衰减率 | 记忆时间尺度 | 捕捉什么 |
|------|--------|-------------|---------|
| h_0 | -1 | 最慢（衰减 ~e^(-t)） | 长期记忆（全局统计） |
| h_1 | -2 | 较快 | 中期变化 |
| h_2 | -3 | 快 | 短期波动 |
| h_3 | -4 | 最快 | 瞬时变化 |

**越低阶（i 越小）的状态，衰减越慢，能记住越远的历史**——这正是 HiPPO 的核心目标：近处历史精细，远处历史粗略。

#### Python 验证

```python
import numpy as np

N = 4
A = np.zeros((N, N))
for i in range(N):
    for j in range(N):
        if i >= j:
            A[i, j] = -(i + 1)

print("HiPPO-LegS 矩阵 A (N=4):")
print(A)

print("\n特征值:")
print(np.linalg.eigvals(A))
# 近似为 [-1, -2, -3, -4]
```

**输出**：
```
HiPPO-LegS 矩阵 A (N=4):
[[-1.  0.  0.  0.]
 [-2. -2.  0.  0.]
 [-3. -3. -3.  0.]
 [-4. -4. -4. -4.]]

特征值:
[-1. -2. -3. -4.]
```

**所有特征值 < 0 → 矩阵稳定** ✅。这就是 HiPPO-LegS 矩阵的设计：每个特征值对应一个独立的衰减通道。

### 6.3.2 为什么下三角？因果结构

**下三角矩阵**意味着：h_i 的更新只依赖于 h_j（j ≤ i）。

**直觉**：
- h_0 只依赖自己（A[0,0] = -1）
- h_1 依赖 h_0 和 h_1（A[1,0] = -2, A[1,1] = -2）
- h_2 依赖 h_0, h_1, h_2（依此类推）

**这是因果结构**：低阶状态捕捉"慢变化"，高阶状态捕捉"快变化"。

### 6.3.3 特征值与稳定性

**HiPPO-LegS 矩阵的特征值**：
```
λ_i = -(i+1)  （近似，对角线值）
```

**例子**（N=4）：
```
λ_0 = -1, λ_1 = -2, λ_2 = -3, λ_3 = -4
```

**稳定性**：所有特征值实部为负 → 稳定 ✓

**物理意义**：
- 越低阶（i 小）的模式，衰减越慢（记住更久）
- 越高阶（i 大）的模式，衰减越快（快速遗忘）

## 6.4 指数级衰减 vs 均匀衰减：为什么重要？

### 两种衰减策略对比

| 衰减方式 | 远处历史权重 | 问题 |
|----------|-------------|------|
| **均匀衰减** | 所有历史权重相同 | 无法区分近远，重要信息被稀释 |
| **指数衰减** | 远处历史权重指数减小 | 远处信息被合理"压缩" |

### HiPPO 的选择：指数级衰减

**指数衰减**（Legendre 多项式在 [0,1] 区间具有此性质）：
```
远的历史：权重 ∝ e^(-λ·t)，λ 较大，快速衰减
近的历史：权重 ∝ e^(-λ·t)，λ 较小，慢速衰减
```

**为什么更好？**

1. **信息瓶颈**：分子性质由"最近的化学环境"决定，远处的原子影响较小
2. **避免梯度消失/爆炸**：稳定的指数衰减保证状态范数有界
3. **数学最优性**：HiPPO 证明了 Legendre 投影是最优的历史压缩（见下文）

**为什么说 Legendre 投影是"数学最优"的？**

这不是随便说的，而是 HiPPO 论文中有严格的数学证明。核心思想是：**HiPPO 把历史压缩问题形式化成了一个优化问题，然后证明了 Legendre 基是这个优化问题的解**。

#### 第一步：把历史压缩定义为一个优化问题

**目标**：用 N 维状态 h(t) 在时刻 t 的值 h_i(t)（i=0,...,N-1），最佳近似历史输入 x(τ) for τ ∈ [0, t]。

**HiPPO 的度量**：定义一个"测量函数"w(τ, t)，衡量在时刻 t 时，历史 τ 有多"重要"：

```
w(τ, t) = (t - τ)^i  （第 i 个基底对应的权重）
```

**优化问题**：找到系数 h_i(t)，使得加权重建误差最小：

```
min_{h_0,...,h_{N-1}} ∫_0^t |x(τ) - Σ_{i=0}^{N-1} h_i(t) · L_i(τ/t)|² · dτ/t
```

这个优化问题的解，就是 Legendre 投影系数。

#### 第二步：为什么 Legendre 是最优的？

**Legendre 多项式的本质是**：在 [0,1] 区间上，以 w(τ) = 1 为权重的正交多项系。

但 HiPPO-LegS 用的是**缩放后的时间** t ∈ [0,1]，做变量替换 u = τ/t：

```
τ ∈ [0, t]  →  u ∈ [0, 1]
```

这样，优化问题变成了在 [0,1] 区间上的标准 Legendre 投影。

**"最优"的具体含义**：

| 最优含义 | 解释 |
|---------|------|
| **L2 意义下误差最小** | 在所有 N 阶多项式基底下，Legendre 系数给出的重建误差最小 |
| **正交性保证无冗余** | 每个 h_i 捕捉独立信息，调整 h_0 不会影响 h_1 |
| **递推关系简单稳定** | HiPPO-LegS 的下三角矩阵结构在数值计算中稳定（条件数最优）|

#### 第三步：更形式化的解释

HiPPO 证明了：**对于在 [0,1] 区间上用多项式近似连续函数的问题，Legendre 基是在 L2 范数意义下的最优正交基**。

这基于以下数学事实：

**Legendre 展开系数的含义**：

```
对于任意连续函数 f(u) on [0,1]，其 Legendre 展开系数：
    h_i = ⟨f, L_i⟩ / ⟨L_i, L_i⟩
其中 ⟨f, g⟩ = ∫_0^1 f(u)·g(u)·du
```

- **⟨f, L_i⟩** = ∫_0^1 f(u)·L_i(u)du — f 和 L_i 的**内积**（类比向量点积 a·b）
- **⟨L_i, L_i⟩** = ∫_0^1 L_i(u)·L_i(u)du — L_i 的"长度平方"（归一化分母，恒正）
- **h_i** — f 在 L_i 方向上的**投影系数**（类比：向量 a 在 x 轴的投影 = a·e_x / |e_x|²）

**部分和 S_N(u)**：

```
S_N(u) = Σ_{i=0}^{N-1} h_i · L_i(u)
```

- 取前 N 项（i=0 到 N-1）的**部分和**
- S_N 是一个 N-1 次多项式
- N=1：S_1(u) = h_0·L_0（常数）；N=2：S_2(u) = h_0·L_0 + h_1·L_1（一次）；N=3：S_3(u) = h_0·L_0 + h_1·L_1 + h_2·L_2（二次）……

**核心收敛定理**：

```
lim_{N→∞} ∫_0^1 |f(u) - S_N(u)|² du = 0
```

- |f(u) - S_N(u)|² 是**平方误差**（非负）
- ∫_0^1 ... du 在 [0,1] 区间上**积分**（求总面积）
- 整个式子说：**当 N→∞ 时，平方误差的总面积 → 0**

**一句话**：用前 N 项 Legendre 多项式加权求和可以任意接近原始连续函数 f，无穷多项时完全一致。这叫做 **L² 收敛**——误差的"能量"趋于零。

**为什么是 L² 而不是逐点收敛？** L² 收敛允许个别点有误差，只要整体平均误差趋于零。这比逐点收敛更弱但更实用——对任意连续 f 都保证，而逐点收敛需要 f 导数连续等额外条件。

**数值例子**：f(u) = e^(-u)，N=4 时部分和 S_4(u) 在 u=0.5 处误差仅 0.0011，∫_0^1 |f-S_4|² du ≈ 0.000018（极小！）。

而 HiPPO 额外证明的是：这个投影系数可以通过一个**简单的常微分方程**（dh/dt = A·h + B·x）在线维护，其中 A 正好是下三角矩阵 A[i,j] = -(i+1) if i ≥ j。

#### 一句话总结"数学最优"

> **HiPPO 的"最优"有两层含义：① Legendre 基在 [0,1] 区间上是 L2 意义下最优的正交基（重建误差最小）；② HiPPO 证明了可以用一个一阶 ODE（dh/dt = A·h + B·x）在线维护这个最优投影系数，而 A 恰好有简单的下三角结构。**

#### 对比：为什么其他基不是最优？

| 基底 | 问题 |
|------|------|
| 幂基 {1, x, x², ...} | 不正交，系数高度耦合（调整一个影响所有） |
| 小波基 | 局部支撑好，但不适合"全局长期依赖"为主的分子历史 |
| Fourier 基 | 周期性假设不适用于分子（边界不闭合） |

Legendre 基同时满足：**正交 + 递推结构简单 + 全局支撑 + 完备**。

> **📌 一句话总结**：HiPPO-LegS 通过下三角矩阵结构实现指数级衰减——低阶模式记住远历史，高阶模式快速衰减，这种结构保证状态压缩是数学最优的。

# Chapter 7: 选择性机制 —— Mamba 的核心创新

## 7.1 标准 SSM vs 选择性 SSM（对比表格）

| 特性 | 标准 SSM（如 S4, DSS） | 选择性 SSM（Mamba） |
|------|----------------------|-------------------|
| **A, B, C** | 固定参数 | 输入相关的函数 |
| **内容选择** | ❌ 无法 | ✅ 可以 |
| **dt** | 固定或标量 | 输入相关 |
| **遗忘机制** | 由 A 矩阵固定控制 | 动态由 dt, B, C 共同控制 |

---

## 7.2 直观理解：像"智能遗忘门"

### 7.2.1 dt = 什么时候更新（时间尺度）

**直觉**：dt 控制"多频繁更新状态"。

```
dt 大 → Δt 大 → 每次迭代变化大 → 快速响应
dt 小 → Δt 小 → 每次迭代变化小 → 平滑/保守
```

**Mamba 的设计**：dt 由输入 x 通过神经网络生成，可以动态调整。

```
dt = softplus(linear_proj(x))
```

**类比**：RL 环境中的"决策频率"——快速变化的环境需要更大的 dt。

### 7.2.2 B = 输入如何影响状态（写入权重）

**直觉**：B 控制"新输入写入状态的强度"。

```
B 大 → 输入强烈影响状态 → 快速记忆新信息
B 小 → 输入微弱影响状态 → 保持旧状态
```

**Mamba 的设计**：B 由输入 x 生成。

```
B = B_proj(x)
```

**类比**：选择性遗忘门——重要的 token（化学键）有更大的 B，被更多记住。

### 7.2.3 C = 状态如何产生输出（读取权重）

**直觉**：C 控制"从状态中读取什么"。

```
C 大 → 状态主导输出
C 小 → 状态微弱影响输出
```

**Mamba 的设计**：C 由输入 x 生成。

```
C = C_proj(x)
```

**类比**：选择性读取门——当前 token 关注状态中的哪些信息。

---

## 7.3 为什么选择能力至关重要？（分子 SMILES 例子）

### 分子 SMILES：CC(=O)OC（乙酸甲酯）

**token 序列**：
```
C   C   (   =   O   )   O   C
位置:0   1   2   3   4   5   6   7
```

**关键观察**：
- `C-C` 单键：重要性一般
- `(=O)` 羰基：化学活性高，需要记住
- `)`：语法标记，重要性低
- `C`（最后）：直链端点，可能不那么重要

### 标准 SSM 的问题

```
所有 token 的 B, C 都相同：
→ 填充 token（语法标记）也被同等记忆
→ 关键官能团 ( = O ) 无法被突出
→ 状态被无关信息"稀释"
```

### 选择性 SSM（Mamba）的优势

```
根据内容生成 B, C：
→ ( = O ) 附近：B 大（强烈写入），C 大（突出读取）
→ 语法标记：C 小（减少影响）
→ 模型自动学习：哪些 token 重要，哪些该遗忘
```

**结果**：更好的分子表示，预测更准确。

> **📌 一句话总结**：选择性机制让 Mamba 能根据输入内容动态决定"记住什么、遗忘什么"，这对于需要聚焦关键信息的分子等任务至关重要。


**图解：Mamba SSM 张量流动（来源：Maarten Grootendorst 博客[^7]）**：

![SSM Architecture - State Equation](https://maartengrootendorst.com/images/posts/2024-21-02-mamba/21.svg)

> 上图：SSM 状态方程——输入 x(t) 通过矩阵 B 影响状态，状态通过矩阵 A 更新自身，状态通过矩阵 C 输出。矩阵 A、B、C 决定系统行为。

![SSM Architecture - Colored](https://maartengrootendorst.com/images/posts/2024-21-02-mamba/23.svg)

> 上图：选择性 SSM（S6）的核心思想——Δ（时间步长）、B（写入权重）、C（读取权重）都变成输入相关的函数，实现内容感知的选择性记忆。

**代码对应**：
```python
# x: (B=2, L=8, d_model=256)
x_proj = x_proj_layer(x)  
# x_proj: (B=2, L=8, d_model + d_state*2) = (2, 8, 384)
# 其中 256 + 64 + 64 = 384
```
```python
# dt: (B=2, L=8, d_model=256) — 离散化步长（每个 token 不同！）
# B_state: (B=2, L=8, d_state=64) — 写入权重（每个 token 不同！）
# C_state: (B=2, L=8, d_state=64) — 读取权重（每个 token 不同！）
dt, B_state, C_state = x_proj.split([d_model, d_state, d_state], dim=-1)
```

**选择性体现在**：
- 不同序列（B 不同）、不同的 token（L 不同），dt、B、C 都不同
- 同一序列中，重要 token 的 dt 向量更大（快速更新），不重要的更小

### 7.4.2 状态更新的选择性

```python
# 标准 SSM（无选择）：
# h_new = exp(dt·A)·h_old + dt·B·x  （dt, B 是标量或固定）

# 选择性 SSM（Mamba）：
# dt, B, C 都是输入的函数，每个 token 不同！
h_new = torch.exp(dt[:,:,None,:] * A) * h_old + dt[:,:,None,:] * B_state[:,:,:] * x[:,:,None]
```

**关键洞察**：选择性是通过**逐元素乘法**实现的——每个 token 有自己的 dt、B、C。

### 7.4.3 选择性计算的具体流程

**完整前向传播步骤**（来自 `bimamba.py` 的实际代码）：

```
输入 x: (B=2, L=8, d_model=256)  ← 分子 SMILES 的 token 序列
```

**Step 1：输入投影 + 分支**

```python
# in_proj: d_model → d_inner*2（d_inner = expand*d_model = 512）
xz = self.in_proj(hidden_states)      # xz: (2, 8, 512)
x, z = xz.chunk(2, dim=-1)             # x 和 z 各: (2, 8, 256)
# x: 用于 SSM 计算
# z: 用于门控（silu gate）
```

**Step 2：因果卷积（局部上下文）**

```python
# conv1d: 捕获局部邻域信息（d_conv=4，核宽度=4）
x = x.transpose(1, 2)                  # (2, 256, 8)
x = self.conv1d(x)[:, :, :seqlen]      # 因果卷积
x = x.transpose(1, 2)                  # (2, 8, 256)
x = self.activation(x)                 # SiLU 激活
```

**Step 3：选择性参数生成（核心！）**

```python
# x_proj: d_inner → dt_rank + d_state*2（x_proj 将 x 投影到选择性参数空间）
x_dbl = self.x_proj(x)                # x_dbl: (2, 8, 256+64+64) = (2, 8, 384)

# split 将向量拆分成三部分
dt, B, C = torch.split(
    x_dbl,
    [self.dt_rank, self.d_state, self.d_state],
    dim=-1
)
# dt:  (2, 8, dt_rank=16)  — 时间步长缩放因子
# B:   (2, 8, d_state=64)  — 写入权重
# C:   (2, 8, d_state=64)  — 读取权重
```

**Step 4：dt 的变换（确保正性 + 有界）**

```python
# dt_proj: dt_rank → d_inner（降维投影）
dt = self.dt_proj(dt)                  # dt: (2, 8, 256)

# softplus: 确保 dt > 0（RNN 的"时间步长"必须为正）
dt = F.softplus(dt)                   # dt: (2, 8, 256)
# softplus(x) = log(1 + exp(x)) ∈ (0, +∞)

# 可选：clamp 防止极端值
dt = torch.clamp(dt, min=-10, max=10)
```

**Step 5：离散化（连续 → 离散）**

```python
# A: (d_state, d_state) = (64, 64)，HiPPO 对角初始化，值为 -1, -2, ..., -64
# A[i,i] = -(i+1)，其他位置为 0

# dA = exp(dt * A)：状态转移矩阵（控制历史信息如何衰减）
# 形状：(B, L, d_state, d_state) = (2, 8, 64, 64)
dA = torch.exp(dt.unsqueeze(-1) * A.unsqueeze(0))

# dB = dt * B：输入到状态的映射（控制新信息如何写入）
# 形状：(B, L, d_state) = (2, 8, 64)
dB = dt.unsqueeze(-1) * B.unsqueeze(1)
```

**Step 6：选择性扫描（并行计算所有时间步）**

```python
# selective_scan: 并行扫描算法，O(L) 复杂度
# 输入：dA, dB, C, x
# 输出：y (B, L, d_state)
y = self.selective_scan(x, dt, B, C)
```

### 7.4.4 选择性如何实现"记住/遗忘"？

**三个选择性参数各自的作用**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        选择性三要素                                  │
├──────────┬──────────────────────────────────────────────────────────┤
│ dt       │ 控制"多长时间更新一次"                                      │
│          │ dt 大 → Δt 大 → 状态快速变化 → 快速响应（记住新信息）         │
│          │ dt 小 → Δt 小 → 状态缓慢变化 → 平滑保守（保留旧信息）         │
├──────────┼──────────────────────────────────────────────────────────┤
│ B        │ 控制"新输入写入多少"                                       │
│          │ B 大 → 输入强烈影响状态 → 快速记忆新信息                      │
│          │ B 小 → 保持旧状态 → 遗忘无关信息                            │
├──────────┼──────────────────────────────────────────────────────────┤
│ C        │ 控制"从状态读取什么"                                       │
│          │ C 大 → 状态主导输出 → 历史信息被重视                         │
│          │ C 小 → 状态微弱影响输出 → 忽略历史                           │
└──────────┴──────────────────────────────────────────────────────────┘
```

**数学上：每个时间步的贡献**

```
y[t] = C[t] · (dA[t] · h[t-1] + dB[t] · x[t])

其中：
  dA[t] = exp(dt[t] * A)  →  历史衰减率（由 dt[t] 调制）
  dB[t] = dt[t] * B[t]   →  新信息写入强度（由 B[t] 调制）
  C[t]                 →  输出选择门（由 C[t] 调制）
```

#### 7.4.4.1 dA 的构成：A 固定，dt 输入决定

**dA 由两部分组成，来源不同**：

```
dA = exp(dt · A)

  ┌─────────────────────────────────────────────────────┐
  │   A（固定）            │   dt（输入决定）            │
  │   HiPPO 初始化          │   softplus(linear_proj(x))  │
  │   决定"怎么衰减"        │   决定"衰减多快"            │
  │   -1, -2, -3...        │   0.1, 0.5, 0.8...        │
  │   相对关系不变           │   随 token 变化             │
  └─────────────────────────────────────────────────────┘
```

**A 的结构是固定的**：

```
A = diag([-1, -2, -3, ..., -64])  ← 初始化后固定

这决定了：
  h_0 的衰减率 = e^(-1·dt)  → 最慢，记住很久以前的事
  h_1 的衰减率 = e^(-2·dt)  → 较快
  h_2 的衰减率 = e^(-3·dt)  → 更快
  ...
  h_63 的衰减率 = e^(-64·dt) → 最快，只关注当前

"h_0 最慢、h_63 最快"这个相对关系由 A 决定，始终不变
```

**dt 由输入决定（调制强度）**：

```
dt = softplus(linear_proj(x[t]))  ← 每个 token 不同

dt 大 → dA 元素接近 1 → 衰减慢 → 记忆更久
dt 小 → dA 元素接近 0 → 衰减快 → 快速更新
```

**具体例子**：

```
分子 "C-C(=O)-O-C" 各位置的 dt：

位置 0: "C"  → dt=0.1  → dA ≈ 0.90 → 快速遗忘
位置 1: "C"  → dt=0.1  → dA ≈ 0.90 → 快速遗忘
位置 2: "="  → dt=0.8  → dA ≈ 0.45 → 慢速遗忘 → 记住更久 ← 重要
位置 3: "O"  → dt=0.5  → dA ≈ 0.61 → 中速遗忘
位置 4: "C"  → dt=0.1  → dA ≈ 0.90 → 快速遗忘
```

**更重要的选择性来自 B 和 C**：

```
dt 的选择性（dA 的调制）只是"辅助的"
B 和 C 的选择性才是"核心的"：

B = linear_proj_B(x[t])  → 决定"写入多少新信息"
C = linear_proj_C(x[t])  → 决定"读取哪些记忆"

→ 重要 token：大 B + 大 C → 强烈记忆
→ 不重要 token：小 B + 小 C → 被忽略
```

#### 7.4.4.2 具体例子：分子 "(=O)" 羰基的选择性记忆

```
假设 token t 是 "="（双键，化学活性高）

x[t] → x_proj → [dt[t], B[t], C[t]]

假设网络学习到：
  dt[t] = 0.8    （大 dt → 快速更新）
  B[t]  = 1.5    （大 B → 强烈写入）
  C[t]  = 1.2    （大 C → 重视状态）

则：
  dA[t] = exp(0.8 * A)  →  衰减较慢（因为 dt 大）
  dB[t] = 0.8 * 1.5    →  强写入

结果："(=O)" 的信息被强烈写入状态，并在后续 token 中被重视。

对比语法标记 ")"：
  dt[t] = 0.1   （小 dt → 几乎不更新）
  B[t]  = 0.2   （小 B → 微弱写入）
  C[t]  = 0.1   （小 C → 忽略状态）

结果：")" 对状态几乎没有影响 → 被"遗忘"
```

### 7.4.5 为什么这个设计能实现内容选择？

**关键洞察：dt, B, C 都是输入的函数**

```
传统的 SSM：dt, B, C 是固定参数 → 对所有输入一视同仁
Mamba：       dt, B, C 由 x 通过神经网络生成 → 每个输入有不同参数

这意味着：模型可以学习"对什么用什么策略"
```

**选择性机制的表达能力**：

```
假设序列长度为 L = 100，d_state = 64

每个 token 有 3 个选择性参数（dt, B, C），每个参数是 d_model 维向量
→ 每个 token 有 3 * d_model = 768 个自由度来控制选择

整个序列的选择性参数总量：L * 3 * d_model = 100 * 768 = 76,800
→ 比固定参数多了 L 倍的表达空间
```

**类比：Transformer 的注意力**

```
Transformer：attention(i,j) = softmax(Q[i] · K[j]) → 选择性地关注任意位置 j
Mamba：       选择性通过 dt, B, C 实现 → 选择性地记住/遗忘不同时间尺度的信息

本质上都是在"决定什么重要，什么不重要"
只是 Mamba 在状态空间做选择（O(N) 复杂度），Transformer 在注意力矩阵做选择（O(N²) 复杂度）
```

# Chapter 8: 并行扫描 —— 从串行到并行

## 8.1 RNN 为什么必须串行？（依赖链）

### RNN 的时间依赖

```
h_0 = f(x_0)                    ← 无依赖，可并行
h_1 = f(x_1, h_0)               ← 必须等 h_0
h_2 = f(x_2, h_1)               ← 必须等 h_1
h_3 = f(x_3, h_2)               ← 必须等 h_2
...
```

**依赖链**：h_t 依赖 h_{t-1}，无法跳过任何一步。

**时间复杂度**：O(N) 串行（虽然每步 O(1)，但必须顺序执行）。

---

## 8.2 洞察：SSM 递归可以展开成结合运算

### 8.2.1 组合算子的推导

**SSM 状态更新**：
```
h_t = dA_t·h_{t-1} + dB_t·x_t
```

**展开来理解（对角 A 简化情况）**：

```
A = diag([-1, -2, -3, -4])           ← d_state = 4 的 HiPPO 对角初始化
dA = exp(dt·A) = diag([exp(-dt), exp(-2dt), exp(-3dt), exp(-4dt)])
dB = dt·B

# 逐维度展开：
h_t[0] = exp(-1·dt) · h_{t-1}[0] + dt · B[0] · x_t
h_t[1] = exp(-2·dt) · h_{t-1}[1] + dt · B[1] · x_t
h_t[2] = exp(-3·dt) · h_{t-1}[2] + dt · B[2] · x_t
h_t[3] = exp(-4·dt) · h_{t-1}[3] + dt · B[3] · x_t
```

**数值例子**（dt=0.1, B=[1,1,1,1], h_0=[0,0,0,0], x_0=1）：

```
dA = diag([0.905, 0.819, 0.741, 0.670])  ← exp(-0.1), exp(-0.2), ...

t=1 时刻：
  h_1[0] = 0.905·0 + 0.1·1·1 = 0.10    ← 全新写入
  h_1[1] = 0.819·0 + 0.1·1·1 = 0.10
  h_1[2] = 0.741·0 + 0.1·1·1 = 0.10
  h_1[3] = 0.670·0 + 0.1·1·1 = 0.10
  → h_1 = [0.10, 0.10, 0.10, 0.10]

t=2 时刻（x_1 = 0.5）：
  h_2[0] = 0.905·0.10 + 0.1·0.5 = 0.0905 + 0.05 = 0.1405  ← 旧记忆衰减 + 新写入
  h_2[1] = 0.819·0.10 + 0.1·0.5 = 0.0819 + 0.05 = 0.1319
  h_2[2] = 0.741·0.10 + 0.1·0.5 = 0.0741 + 0.05 = 0.1241
  h_2[3] = 0.670·0.10 + 0.1·0.5 = 0.0670 + 0.05 = 0.1170
  → h_2 = [0.14, 0.13, 0.12, 0.12]

时间线：
  t=0: h_0 = [0,    0,    0,    0   ]  ← 初始状态
        ↓ x_0=1
  t=1: h_1 = [0.10, 0.10, 0.10, 0.10]  ← 记录了 x_0 的信息
        ↓ x_1=0.5
  t=2: h_2 = [0.14, 0.13, 0.12, 0.12]  ← 旧记忆衰减 + 新输入
        ↓ ...
```

**关键洞察**：每个记忆槽 i 独立更新：h_t[i] = dA[i,i] · h_{t-1}[i] + dB[i] · x_t。A 是对角矩阵，所以没有"槽之间的干扰"——每个槽只管自己的衰减速度。

**展开**（k < t）：
```
h_t = dA_t·h_{t-1} + dB_t·x_t
     = dA_t·(dA_{t-1}·h_{t-2} + dB_{t-1}·x_{t-1}) + dB_t·x_t
     = dA_t·dA_{t-1}·h_{t-2} + dA_t·dB_{t-1}·x_{t-1} + dB_t·x_t
     = (dA_t·dA_{t-1})·h_{t-2} + (dA_t·dB_{t-1} + dB_t)·x_{t-1} + ...
```

**观察到结合性**：
```
h_t = (dA_t, dB_t·x_t) ⊕ (dA_{t-1}, dB_{t-1}·x_{t-1}) ⊕ ... ⊕ (dA_0, dB_0·x_0) ⊕ h_init
```

**组合算子 ⊕ 定义**[^6]：
```
(A, b) ⊕ (A', b') = (A·A', A·b' + b)
```

**验证**：
```
(A, b) ⊕ (A', b')
= (A·A', A·b' + b)  ← 这正是展开后的形式！
```

## 8.3 数值例子：3步序列，手算每种方式的结果对比

### 设定

```
dA_0 = 0.5, dB_0 = 0.1, x_0 = 1, h_init = 1
dA_1 = 0.4, dB_1 = 0.2, x_1 = 1
dA_2 = 0.3, dB_2 = 0.3, x_2 = 1
```

### 方式 1：串行 RNN

```
h_0 = dA_0·h_init + dB_0·x_0 = 0.5·1 + 0.1·1 = 0.6
h_1 = dA_1·h_0 + dB_1·x_1 = 0.4·0.6 + 0.2·1 = 0.44
h_2 = dA_2·h_1 + dB_2·x_2 = 0.3·0.44 + 0.3·1 = 0.432
```

### 方式 2：组合算子并行

**第一步：计算所有 (A_i, b_i)**：
```
(A_0, b_0) = (dA_0, dB_0·x_0) = (0.5, 0.1·1) = (0.5, 0.1)
(A_1, b_1) = (0.4, 0.2·1) = (0.4, 0.2)
(A_2, b_2) = (0.3, 0.3·1) = (0.3, 0.3)
```

**第二步：并行扫描**（类似前缀和）：



```
level by level）：
Level 0: [0] [1] [2]  ← 原始
Level 1: [0⊕1] [2]    ← combine(0,1)
Level 2: [0⊕1⊕2]      ← combine((0⊕1), 2)

# Level 1
s0 = (A_0, b_0) ⊕ (A_1, b_1) = (0.5·0.4, 0.5·0.2 + 0.1) = (0.2, 0.2)
s1 = (A_2, b_2)

# Level 2
s_comb = s0 ⊕ s1 = (0.2·0.3, 0.2·0.3 + 0.2) = (0.06, 0.218)

h_2 = A_comb·h_init + b_comb = 0.06·1 + 0.218 = 0.278
```

**组合算子的验证**：
```
h_t = A_t·h_{t-1} + b_t
     = A_t·(A_{t-1}·h_{t-2} + b_{t-1}) + b_t
     = A_t·A_{t-1}·h_{t-2} + A_t·b_{t-1} + b_t

对比：
(A_t, b_t) ⊕ (A_{t-1}, b_{t-1}) = (A_t·A_{t-1}, A_t·b_{t-1} + b_t) ✓
```

**所以组合顺序应该是从新到旧**：
```
h_t = (A_t, b_t) ⊕ (A_{t-1}, b_{t-1}) ⊕ ... ⊕ (A_0, b_0) ⊕ h_init
```

**正确重算**（反向组合）：
```
(A_2, b_2) ⊕ (A_1, b_1) = (0.3·0.4, 0.3·0.2 + 0.3) = (0.12, 0.36)
(0.12, 0.36) ⊕ (A_0, b_0) = (0.12·0.5, 0.12·0.1 + 0.36) = (0.06, 0.372)

h_2 = 0.06·1 + 0.372 = 0.432 ✓
```

> **📌 一句话总结**：SSM 的递归可以通过组合算子 (A,b) ⊕ (A',b') = (A·A', A·b' + b) 转化为结合运算，这使得并行扫描（非串行）成为可能，最终 O(N) 步完成而非 O(N) 深度串行。

## 8.4 并行扫描 vs 串行 RNN vs 卷积（复杂度对比表）

| 方式 | 时间复杂度 | 空间复杂度 | 并行化 | 因果性 |
|------|-----------|-----------|--------|--------|
| **串行 RNN** | O(N) | O(1) | ❌ | 天然因果 |
| **卷积 SSM** | O(N log N) | O(N) | ✅ | 需掩码 |
| **并行扫描** | O(N) | O(N) | ✅ | 天然因果 |

**为什么并行扫描更快？**

- 串行 RNN：O(N) 深度 = 如果 N=1000，需要 1000 步顺序执行
- 并行扫描：O(N) 宽度 = 利用 GPU 并行能力，同时计算所有层级

## 8.5 硬件感知设计：为什么重要？（HBM vs SRAM）

### GPU 内存层级

```
HBM（High Bandwidth Memory）：
- 大容量（8-16 GB）
- 带宽高但延迟大
- 需要频繁读写

SRAM（Static RAM）：
- 小容量（~100 MB per thread block）
- 带宽极高，延迟极低
- 昂贵

问题：中间结果太多，HBM 带宽成为瓶颈
```

### 融合 Kernel 解决方案

**融合（Fusion）**：把多个操作合并成一个，减少 HBM 读写。

```
标准实现：
h1 = read(HBM) → compute → write(HBM)
h2 = read(HBM) → compute → write(HBM)
... （1000 次 HBM 读写！）

融合实现：
一次 read → compute all steps → 一次 write
（HBM 读写减少 1000 倍）
```

**Mamba 的实现**：使用 CUDA 融合 kernel，避免中间结果写回 HBM，直接在 SRAM 完成扫描。

> **📌 一句话总结**：并行扫描虽然理论上 O(N)，但实际需要硬件感知的融合实现避免 HBM 带宽瓶颈——这是 Mamba 高效的关键工程技巧。

# Chapter 9: Bi-Mamba —— 双向建模

## 9.1 为什么需要双向？（前向 vs 后向信息）

### 单向 SSM 的局限

**前向 SSM**：
```
x_0 → x_1 → x_2 → x_3 → ... → x_N
```

**问题**：预测 x_3 时，只能看到 x_0, x_1, x_2（过去信息），看不到 x_4, x_5, ...（未来信息）。

### 为什么这很重要？

**分子 SMILES**：`c1ccccc1`（苯环）

```
位置:  0  1  2  3  4  5  6
Token: c  1  c  c  c  c  1

"1" 是环闭合标记，需要和前面的 "c" 配对
→ 单向模型可能无法正确理解环结构
→ 双向模型可以从两边同时看
```

**其他场景**：
- DNA 序列：互补链需要双向信息
- 蛋白质：远距离氨基酸相互作用
- 自然语言：有些词需要看上下文才能理解

## 9.2 结构：前向层 + 后向层 + 融合门

### Bi-Mamba 架构

```
输入序列
    │
    ├──→ [Forward Mamba Layers] → forward_h
    │                              (左→右处理)
    │
    └──→ [Backward Mamba Layers] → backward_h
         (右→左处理) ← 输入反转
              │
              ↓
         backward_h 反转回正常顺序
              │
    ┌─────────┴─────────┐
    ↓                   ↓
[forward_h]     [backward_h]
    │                   │
    └────────┬──────────┘
             ↓
      [Fusion Gate]
             │
             ↓
       fused_output
```

### 融合门控

**融合门决定**：每个位置应该多依赖前向，多依赖后向？

```python
# 融合门计算
combined = torch.cat([forward_h, backward_h], dim=-1)
gate = torch.sigmoid(self.fusion_gate(combined))
gate_f, gate_b = gate.chunk(2, dim=-1)

fused = gate_f * forward_h + gate_b * backward_h
```

---

## 9.3 融合门控机制的物理意义

### 门控 = 软性选择

**无融合门**（简单平均）：
```
fused = (forward_h + backward_h) / 2
```
问题：所有位置同等依赖前向和后向。

**有融合门**：
```
fused = gate_f * forward_h + gate_b * backward_h
```
- gate_f, gate_b ∈ [0, 1]
- gate_f + gate_b = 1（理论上）
- **每个位置动态决定前向/后向的比重**

### 物理直觉

**分子开头**（位置 0）：
- 可能需要更多后向信息（看分子后面是什么）
- gate_b 较大

**分子结尾**（位置 N）：
- 可能需要更多前向信息（看分子前面是什么）
- gate_f 较大

**化学键中心**：
- 可能需要平衡前向和后向
- gate_f ≈ gate_b

# Appendix A: 公式速查表

## A.1 连续时间 SSM

```
dh/dt = A·h + B·x        （状态更新微分方程）
y     = C·h               （输出方程）
```

## A.2 精确解（积分形式）[^1]

```
x(t_b) = e^(A·(t_b-t_a))·x(t_a) + ∫_{t_a}^{t_b} e^(A·(t_b-τ))·B·u(τ)·dτ
```

## A.3 离散化方法

| 方法 | A_d | B_d |
|------|-----|-----|
| Forward Euler[^2] | `I + Δt·A` | `Δt·B` |
| Backward Euler | `(I - Δt·A)⁻¹` | `(I - Δt·A)⁻¹·Δt·B` |
| Bilinear[^3] | `(I - Δt·A/2)⁻¹·(I + Δt·A/2)` | `(I - Δt·A/2)⁻¹·Δt·B` |
| 指数映射 | `exp(Δt·A)` | `Δt·B` |

## A.4 s-z 映射

| 方法 | 映射公式 |
|------|---------|
| 理想 | `z = e^(sT)` |
| Forward Euler[^2] | `z = 1 + sT` |
| Backward Euler | `z = 1/(1-sT)` |
| Bilinear[^3] | `z = (1+sT/2)/(1-sT/2)` |

## A.5 Mamba 离散化

```
dA = exp(dt·A)     ← 矩阵指数
dB = dt·B          ← 简化近似
```

## A.6 选择性机制

```
dt = f(x)          ← 离散化步长（输入相关）
B  = g(x)          ← 写入权重（输入相关）
C  = h(x)          ← 读取权重（输入相关）
```

## A.7 并行扫描组合算子[^6]

```
(A, b) ⊕ (A', b') = (A·A', A·b' + b)
```
